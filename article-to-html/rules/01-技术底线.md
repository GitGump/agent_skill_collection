# 技术底线

> 以下是经过实际测试验证的硬性要求。**设计可以自由发挥，但技术底线不可突破。**

## 小红书 3:4 比例规范

| 参数 | 值 | 说明 |
|------|----|------|
| 画布宽度 | 1080px | 固定值 |
| 切片高度 | 1440px | 每张切片的固定高度（3:4 = 1080:1440） |
| 完整长图 | 1080 × N×1440 | N 为切片数量，总高度固定为 N 的整数倍 |
| 输出文件 | `xxx_full.png` + `xxx_slice_01.png` ... | 完整长图 + 各切片 |

## 必须包含的 CSS

```css
* { margin: 0; padding: 0; box-sizing: border-box; }
body { margin: 0; padding: 0; background: transparent; }

.container {
  width: 1080px;        /* 固定宽度 */
  margin: 0;
  padding: 0;
}

/* 每张切片的容器 — 核心结构 */
.slice-card {
  width: 1080px;
  height: 1440px;       /* 固定高度，不用 min-height */
  padding: 16px 48px;   /* 默认中间切片：极简留白，最大化内容区 */
  box-sizing: border-box;
  overflow: hidden;      /* 内容不得溢出卡片，设计时必须控制内容量 */
  position: relative;
  display: flex;         /* flex 布局让内容自动分配空间 */
  flex-direction: column;
}

/* 差异化留白策略 — 关键优化！ */
.slice-card.slice-first {
  padding: 60px 48px;   /* 首切片：宽松留白，大标题吸引注意力 */
  justify-content: center; /* 首切片内容居中 */
}
.slice-card.slice-mid {
  padding: 16px 48px;   /* 中间切片：极简留白，内容尽可能饱满 */
  justify-content: space-between; /* 中间切片内容自动上下分散填满 */
}
.slice-card.slice-last {
  padding: 48px 48px;   /* 尾切片：适中留白，收尾有呼吸感 */
  justify-content: flex-start; /* 尾切片从顶部排列，底部自然留白 */
}
```

> ⚠️ **`.slice-card` 的 `height: 1440px` 是固定值，不是 min-height。** 这确保了总高度精确等于 N × 1440px，切片时不会截断内容。

### 差异化留白的设计意图

| 位置 | 上下padding | 可用内容高度 | flex布局 | 为什么 |
|------|-----------|------------|---------|--------|
| 首切片 `.slice-first` | 上下各60px | 1320px | justify-content:center | 大标题+钩子信息居中展示，视觉冲击力强 |
| 中间切片 `.slice-mid` | 上下各16px | 1408px | justify-content:space-between | 内容为王，flex自动分配空间填满，**绝不留底部空白** |
| 尾切片 `.slice-last` | 上下各48px | 1344px | justify-content:flex-start | 收尾从顶部排列，底部自然留白给回味空间 |

> **拼接时长图的留白间距**：首→中 60+16=76px，中→中 16+16=32px，中→尾 16+48=64px。节奏紧凑但不突兀。

### ⚠️ 中间切片的 flex 布局是核心优化

中间切片使用 `display:flex; flex-direction:column; justify-content:space-between`，这意味着：
- 内容元素（标题、列表、卡片、引用等）会**自动均匀分配**到切片的整个高度
- **不会出现底部大片空白**——即使内容没有精确占满，flex 也会拉开元素间距
- 设计时只需关注内容是否合理，不需要手动计算高度

### 首切片设计建议

首切片是小红书信息图最重要的切片——它决定观众是否继续看。设计时应：
- **大标题占视觉中心** — 字号 84-128px，1-2 行核心标题
- **钩子信息** — 2-3 个关键数据/对比，激发好奇
- **场景引入** — 一段简短描述把读者拉进故事
- **避免信息过载** — 首切片宁留白也不要塞满，重点是视觉冲击力
- **不需要"右滑查看"之类的提示** — 视觉设计本身就是最好的引导

### 留白的作用

1. **单独看** — 每张切片有呼吸感，内容不顶边
2. **拼成长图** — 相邻切片的留白合并为自然间隔，形成节奏感
3. **过渡缓冲** — 即使切片间有微妙的背景色变化，留白区域能平滑过渡

### 切片间视觉连续（可选进阶）

如果需要切片间视觉无缝衔接，可以在 `.slice-card` 上添加：

```css
/* 方案 A：底部渐变淡出 + 顶部渐变淡入 */
.slice-card::after {
  content: '';
  position: absolute;
  bottom: 0; left: 0; right: 0;
  height: 48px;
  background: linear-gradient(to bottom, transparent, var(--slice-bg, #fff));
}
.slice-card::before {
  content: '';
  position: absolute;
  top: 0; left: 0; right: 0;
  height: 48px;
  background: linear-gradient(to top, transparent, var(--slice-bg, #fff));
  z-index: 1;
}

/* 方案 B：简单的分割线 */
.slice-card + .slice-card {
  border-top: 1px solid rgba(0,0,0,0.06);
}
```

> 选择哪种方案取决于视觉风格。报纸风适合分割线，深色科技风适合渐变。

## HTML 结构模板

```html
<body>
  <div class="container">
    <div class="slice-card slice-first" data-slice="1">
      <!-- 首切片：大标题 + 钩子信息 + 场景引入 -->
    </div>
    <div class="slice-card slice-mid" data-slice="2">
      <!-- 中间切片：内容饱满 -->
    </div>
    <div class="slice-card slice-mid" data-slice="3">
      <!-- 中间切片：内容饱满 -->
    </div>
    <div class="slice-card slice-last" data-slice="4">
      <!-- 尾切片：收尾升华 -->
    </div>
  </div>
</body>
```

> **每个 `.slice-card` 就是一张独立的信息图卡片。** 设计时把它当作独立的 1080×1440 画布来思考。
> **必须标注位置类**：`.slice-first`、`.slice-mid`、`.slice-last`，用于差异化 padding。

## 字号（1080px 画布，手机缩放 ~3x）

| 元素 | 最小 | 推荐 |
|------|------|------|
| H1 主标题 | 84px | 100px |
| H2 模块标题 | 48px | 52px |
| 正文 | 36px | 38-40px |
| 关键数据 | 72px | 80px |
| 标签/脚注 | 32px | 34px |

## 布局（每张切片内部）

- 首切片可用内容区 = 1080×1320px（上下各60px留白，内容居中）
- 中间切片可用内容区 = 1080×1408px（上下各16px留白，flex自动分配）**← 最大的内容区**
- 尾切片可用内容区 = 1080×1344px（上下各48px留白，从顶部排列）
- 最多 **2 栏**，禁止 3 栏以上
- 单栏优先，双栏仅用于并列对比
- **每张切片 2-4 个核心模块**（中间切片可放最多4个，首尾切片2个）
- 每张切片内容尽可能丰富的同时保留适当的留白
- 模块间距 gap ≥ 20px
- **总切片数 4-8 张**，不要超过 8 张

## 内容量控制（关键！）

每张切片的内容量必须控制在可用高度内，**中间切片 flex 布局会自动分配空间**：

| 切片位置 | 可用高度 | 建议内容量 |
|----------|---------|-----------|
| 首切片 | 1320px | 大标题(~200px) + 数据条(~120px) + 场景引入(~100px) |
| 中间切片 | 1408px | 尽可能多放！2-4个模块，flex自动拉开间距填满 |
| 尾切片 | 1344px | 核心内容(~600px) + 收尾金句/结语区域，底部自然留白 |

| 内容类型 | 一张中间切片能放多少 |
|----------|---------------|
| 大标题 + 数据条 + 3格卡片 | ✅ 刚好填满 |
| 时间线 5-6 个节点 | ✅ 合适（紧凑排列） |
| 对比表格 5-6 行 | ✅ 合适 |
| 框架卡片3个 + 对比表格 | ✅ 紧凑但可行 |
| 段落正文超过 8 行 | ⚠️ 可能溢出，考虑精简 |

> **中间切片的黄金法则：内容尽可能饱满，flex 布局自动分配空间，绝不留底部空白。**
> **首切片内容居中，形成视觉冲击力。** 尾切片从顶部排列，底部自然留白给回味空间。

## 禁止

| 写法 | 后果 |
|------|------|
| `.slice-card { height: auto }` | 总高度不确定，切片边界对不齐 |
| `.slice-card { min-height: 1440px }` | 内容少时底部空白，切片不精确 |
| `body { padding: 40px }` | 四周大白边 |
| `.container { max-width: 800px; margin: 0 auto }` | 两侧空白 |
| `.slice-card { overflow: visible }` | 内容溢出到相邻切片 |
| CSS columns | 手机上栏太窄 |
| 正文 < 36px | 手机上看不清 |
| 统一 `padding: 72px 48px` | 中间切片留白过多，长图出现大片空白 |
| 中间切片底部空白 | 没有 flex 布局或内容不足导致底部留空 |
| "右滑查看" "swipe-hint" 之类的提示 | 视觉设计本身就是引导，不需要文字提示 |
