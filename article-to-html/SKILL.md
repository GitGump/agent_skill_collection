# 文章转 HTML 信息图

文章 → 提炼核心信息 → 切片单元设计 → 生成 HTML → 后处理 → 截图交付。

## 设计理念

**你是设计师，不是模板填充机。** 根据文章内容自由设计视觉风格，不要每次都用同一个模板。

- **内容决定形式** — 严肃分析用深色/衬线，轻松科普用手账/波普，技术教程用终端/蓝图
- **信息精炼** — 每张切片 2-4 个核心模块，首尾切片 2 个、中间切片可放最多 4 个，不是把文章全文塞进去
- **手机第一** — 图片在手机上缩放 3 倍，字号必须够大（详见 `rules/01-技术底线.md`）
- **每次不同** — 配色、字体、布局每次都应该有变化，避免千篇一律

## 核心方法：切片先行设计

**先想切片，再写 HTML。** 不是先生成长页面再切割，而是从设计阶段就以每张切片为独立设计单元。

### 旧方法的问题 ❌
```
长 HTML → 截长图 → 机械切片 → 切片边界处内容被截断
```

### 新方法 ✅
```
规划切片内容 → 每张切片独立设计 → HTML 中每张切片是一个 .slice-card → 拼接成长图 → 完美切片
```

### 切片先行三原则

1. **先分块再填内容** — 先决定要几张切片，每张切片讲什么，再往里面填内容
2. **每张切片自洽** — 每张切片的视觉和内容都相对完整，不依赖上下切片才能理解
3. **差异化留白** — 不同位置的切片留白策略不同，让长图整体节奏舒适：
   - **首切片** — 大标题 + 钩子信息，内容居中，留白多一些
   - **中间切片** — 内容尽可能饱满丰富（最多4个模块），flex 布局自动分配空间填满，同时保留适当留白，**绝不留底部空白**
   - **尾切片** — 收尾升华，从顶部排列，底部自然留白给回味空间

### 切片内容规划示例

以一篇人物故事为例，规划 4 张切片：

| 切片 | 位置 | 内容定位 | 设计要素 | 留白策略 |
|------|------|---------|---------|---------|
| 切片 1 | 首 | 开篇钩子：大标题 + 核心数据 + 引入场景 | 大字标题、数据条、场景图 | 宽松（上下60px，居中） |
| 切片 2 | 中 | 主体叙事：关键转折点 + 核心论据 | 时间线、对比表格、引用 | 极简（上下16px，flex填满） |
| 切片 3 | 中 | 深度展开：支撑证据 + 数据/细节 | 数据卡片、引用、框架图 | 极简（上下16px，flex填满） |
| 切片 4 | 尾 | 收尾升华：金句 + 结语 | 大引用、结语、深色背景收尾 | 适中（上下48px，顶排） |

## 工作流程

```
用户给文章 + 文章标题
    ↓
① 内容分析 — 提取核心论点、关键数据、对比维度、避坑要点、金句
    ↓
② 切片规划 — 决定切片数量（4-8张），每张切片的内容定位和模块
              标注每张切片的位置角色（首/中/尾）和留白策略
    ↓
③ 视觉设计 — 根据内容调性，自主决定视觉风格、配色、布局
              （可参考 rules/03-风格灵感.md 和 templates/ 获取灵感）
    ↓
④ 生成 HTML — 自包含 HTML，必须遵守 rules/01-技术底线.md
              每个 .slice-card 根据位置使用差异化 padding
    ↓
⑤ 创建版本文件夹（强制）—
              运行 post-process.sh 时传入标题：
              bash post-process.sh <html文件> <文章标题> [输出目录]
              → 自动创建「标题-时间戳-V版本号」文件夹（版本号递增）
              → 产物全部保存在该文件夹内
              默认输出目录：E:/codefile/trae/mp_article/文生图/
    ↓
⑥ 后处理（自动化）—
              post-process.sh 自动完成：fix-html.js 修复 + 字号兜底注入
    ↓
⑦ 截图交付 — 按 rules/02-截图流程.md 执行
              → 总高度 = N × 1440px，无需精确测量
              → 将截图保存到版本文件夹内（命名 index_full.png）
    ↓
⑧ 切片生成 —
              node scripts/slice-image.js <完整长图.png> --out <版本文件夹>
              → 自动生成 index_full.png + index_slice_01.png, _02.png...（小红书切片）
```

**步骤⑤⑧ 都是强制的。** 每次生成完整流程必须产出：
- 1 个版本文件夹（命名：`{标题}-{YYYYMMDDHHMM}-V{N}`）
- 1 张完整长图
- N 张小红书 3:4 切片图

### 版本号管理

版本号在技能目录 `~/.workbuddy/skills/article-to-html/.version_counter` 中递增管理。
每次运行自动 +1，支持并发安全（flock 锁）。
同一天内同一标题的多个版本会依次生成 V1、V2、V3……

### 产物文件夹结构

```
E:\codefile\trae\mp_article\文生图\
└── {标题}-{时间戳}-V版本号\
    ├── .meta.txt           ← 元信息（标题/时间戳/版本号）
    ├── index.html          ← 后处理后的 HTML
    ├── index_full.png       ← 完整长图（需手动截图）
    └── index_slice_01.png  ← 小红书切片图 × N张
```
