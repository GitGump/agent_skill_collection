# Skill: 产研内部评优颁奖词生成器

## 技能描述
根据用户输入的员工姓名、获奖荣誉（质量之星/创新之星/AI先锋之星）、角色（仅质量之星需要）、获奖理由，自动生成符合公司产研风格的精简颁奖词，并输出完整荣誉证书文案。

## 触发条件
用户提供以下信息：
- 姓名（必填）
- 获奖荣誉（质量之星 / 创新之星 / AI先锋之星）
- 角色（仅当获奖荣誉为“质量之星”时必填，可选：前端开发、后端开发、测试、产品、设计）
- 获奖理由（一段或多条要点，自由文本）

## 输出格式
严格按照以下结构输出：

```
**荣誉证书**

**{姓名} 同事：**

{根据角色和理由生成的事迹描述，精简、突出亮点}

特此授予你 **“{获奖荣誉}”** 荣誉称号。

{适合该荣誉和角色的口号，单独一行}

感谢你在产研一线的{对应贡献描述}！

**技术管理委员会**  
{当前年月，格式：二〇二五年四月}
```

## 生成步骤

1. **解析输入**：提取姓名、获奖荣誉、角色（质量之星时需要）。获奖理由文本可自由格式，可能包含编号、断句等。
2. **精简理由**：将长篇理由提炼为2-4个短句，保留关键数据（单量、百分比、数量等），去掉冗余修饰，使用动作+成果的句式。例如：“完成xx”、“实现xx”、“提升xx”、“保障xx”。
3. **选择模板**：
   - 若为“质量之星”：根据角色匹配对应的口号、尾句和能力关键词，并生成事迹描述。
   - 若为“创新之星”：使用统一口号和尾句，事迹描述聚焦创新点。
   - 若为“AI先锋之星”：使用统一口号和尾句，事迹描述突出AI辅助、提效、工程落地和可推广性。
4. **生成日期**：当前年月，使用中文数字，如“二〇二六年三月”。建议动态获取，若无则默认“二〇二六年四月”。
5. **输出完整证书**：按格式拼接。

## 注意事项
- 对于质量之星，如果用户输入的角色不在五种预设中（如“运维”、“数据”等），默认使用“后端开发”或由用户指定最接近角色，或给出提示。
- 事迹描述必须基于用户提供的理由，不能凭空捏造；可适当合并同类项、精简冗余。
- 保持语言简洁、有力，避免“的”、“了”等多余助词。
- 数字建议保留，如“5万张”、“450单”、“40.75%”等。
- 日期动态生成：建议使用当前系统时间，转换为中文数字年月。如无法获取，默认“二〇二六年四月”。
