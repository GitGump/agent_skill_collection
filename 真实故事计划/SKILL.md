# 真实故事计划 —— 微信公众号非虚构写作助手

## 任务目标
本 Skill 用于：模仿「真实故事计划」风格，快速生成高质量非虚构类微信公众号文章，从选题策划、故事创作、配图生成到排版发布的一站式服务。

能力包含：
1. 挖掘符合「真实故事计划」调性的选题
2. 以第一人称叙事为主，创作 **2000～5000 字**的非虚构风格公众号文章
3. 自动生成正文配图和封面图（纪实感、人文气质）
4. 自动完成文章排版（适合公众号展示，克制、留白、沉浸）
5. 生成可直接复制到公众号的 HTML 文件
6. 提供发布操作指引

触发条件：
- 用户说「帮我写一篇真实故事计划风格的文章」
- 用户提供一个真实故事线索/采访素材/生活观察
- 用户说「写一篇非虚构」「写一个普通人的故事」「写底层叙事」
- 用户提到职场、情感、边缘群体、乡土中国、城市漂泊、生死、疾病、犯罪、小众职业等主题

## 操作步骤

### 标准流程

**步骤1：选题与采访素材梳理**

基于用户提供的线索/主题，智能体将：

1. **判断主题适配性**：
   - 是否属于真实故事计划核心选题范围：职场、情感、边缘群体、乡土中国、城市漂泊、生死、疾病、犯罪、小众职业
   - 评估故事中是否有真实的人物、具体的处境、情感的流动
   - 确认：是否有「人性的挣扎与微光」——这是最核心的价值判断

2. **提炼核心叙事角度**：
   - 这个故事从谁的视角讲？（第一人称叙述者）
   - 这个人面对的核心困境是什么？
   - 故事最打动人的细节/时刻在哪里？
   - 不追求猎奇，而是挖掘**普通处境中的非凡体验**

3. **建立故事结构骨架**：
   - 开场：一个具体的场景、细节或动作，直接抛入故事中
   - 背景铺垫：主人公是谁，她/他从哪里来
   - 核心事件：改变发生的那个节点
   - 情感流动：内心世界的挣扎与选择
   - 余韵结尾：不说教，不升华，留白给读者

**选题参考**：见 [references/story-topic-guide.md](references/story-topic-guide.md)

**步骤2：文章创作**

以「真实故事计划」写作风格创作文章：

**写作风格核心特征**：
- **叙述视角**：第一人称为主，强调「我」的感受、困惑、选择
- **语言质地**：真实、克制、有质感，不刻意煽情，情绪在细节中渗透
- **叙事节奏**：慢进、不赶，细节密度高，给读者沉浸感
- **故事姿态**：不俯视，不仰视，与笔下的人同等高度站立
- **情感原则**：不说教，不定性，让读者自己感受
- **禁止词汇**：禁用「充满正能量」「逆袭成功」「感动了无数人」等廉价赞美
- **结尾处理**：开放式结尾，留有余地，让现实本身说话

**代表性叙事技法**：
- 文章开头用**采访口述引言**交代主人公简要信息（详见下方「口述引言」要求）
- 正文开篇用一个极具现场感的细节/场景切入（不再在正文中用「我叫XXX，今年XX岁」自我介绍）
- 用具体动作代替情绪描述（「她把碗推到一边」而非「她很难过」）
- 穿插对话和内心独白，增加人物厚度
- 避免连续议论，让故事本身承载意义
- 文章标题：朴素有力，像一句陈述句或一个问题，不用感叹号

**口述引言（开头段落）**：
- 位于标题与封面图之后、正文之前
- 以记者采访的口吻撰写，参考「真实故事计划」「人物」杂志的引言风格
- **引言内容应包含**：
  - 主人公基本信息：姓名（化名）、年龄、籍贯/家乡
  - 职业/身份：当前或核心的职业状态
  - 故事背景：主人公所处的具体处境、面临的困境或关键事件
  - 一个具体的细节或场景暗示：让读者在引言中就能感受到故事的氛围
  - 采访过渡语：用"以下是他的讲述""本文根据她的讲述整理"等方式引入正文
- **引言篇幅**：3～6 句话，比纯信息交代稍丰富，但仍保持克制
- **写作原则**：
  - 不做评价，不用形容词堆砌（避免"坚强地""勇敢地"）
  - 用事实和细节说话，而非抽象概括
  - 让读者通过引言就能理解故事发生的来龙去脉
  - 最后一句必须是采访过渡语，自然引出正文
- **示例**：
  - 短版（3句）：「周洋，31岁，安徽芜湖人。他在一家互联网大厂做了三年数据分析师，直到2025年冬天，公司开始用AI工具替代部分数据岗位。以下是他的讲述。」
  - 丰富版（5句）：「李芸，28岁，湖南衡阳人。2019年她跟着老乡来到北京，注册成了外卖骑手。四年里她跑坏了三辆电动车，送了将近四万单，膝盖落下了旧伤。今年三月，站点通知骑手收入体系调整，她算了笔账，决定回老家。这篇文章根据她的讲述整理。」
  - 人物杂志风格：「陈卫国，54岁，重庆万州人。他在长江大桥上做了十二年的巡桥员，每天走过两万步，劝下过十七个想跳桥的人。去年底，巡桥队被纳入智能监控系统试点，他签了一份新的合同，岗位名称变成了'人机协同巡查员'。以下是他的讲述。」
- **Markdown 格式**：使用引用语法 `>` 包裹，例如：
  ```
  > 周洋，31岁，安徽芜湖人。他在一家互联网大厂做了三年数据分析师，直到2025年冬天，公司开始用AI工具替代部分数据岗位。以下是他的讲述。
  ```
- **样式要求**：这段文字在 HTML 中使用**浅灰色、小号字体**，与正文明确区分（generate_html.py 已内置对应样式）

1. **生成文章结构**：
   - 字数：**2000～5000 字**（可根据素材量灵活调整，口述引言不计入正文字数）
   - 分 4～6 个自然段落，段与段之间有情节推进
   - 每个段落围绕一个时间节点或情绪转折

2. **撰写正文内容**：
   - 口述引言：用 `>` 引用格式写一段主人公简要信息+采访过渡语
   - 开篇：抛入具体场景（地点+时间+人物动作），3～5 句内建立现场感
   - 中段：故事的主要进程，穿插细节、对话、内心独白
   - 转折点：那个改变了一切的时刻，是全文情感密度最高处
   - 收尾：带着未解的余味，不强行给出答案

3. **优化润色**：
   - 检查是否有「流量写作」套路痕迹（如「扎心了」「破防了」「泪崩」）
   - 确认细节足够具体，避免泛泛而谈
   - 语言节奏是否有轻重缓急，避免每句话情绪密度相同
   - 检查结尾是否说得太透

4. **生成标题与文件名**（文章完成后）：
   - 基于全文内容，提炼核心叙事线索
   - 标题风格：**朴素、直白、有重量**，像一个陈述事实的句子
   - **文件名必须带时间戳**：格式为 `{标题}-{YYYYMMDDHHmm}`
     - 例如：`赶时间的人-202604171148`、`长江大桥上的守桥人-202604201530`
     - 时间戳取当前系统时间（精确到分钟）
     - **目的**：同一主题多次生成的文章不互相覆盖
   - 标题参考：
     - 《赶时间的人》（简洁，有多义性）
     - 《我在奥运会上做黄牛》（第一人称，具体职业/场景）
     - 《被资助长大的失足女孩》（人物状态，引发追问）
     - 《长江大桥上的守桥人》（具体地点+角色）
   - 生成 3～5 个备选标题，优先选最朴素的那个
   - 避免：感叹号、过度悬念词（「震惊」「绝了」「真相」）

**标题生成参考**：见 [references/story-topic-guide.md](references/story-topic-guide.md)

**步骤3：配图生成与插入**

智能体将为文章生成配图，风格与内容相符：

1. **生成封面图**：
   - 风格定位：**纪实感、人文气质、克制不猎奇**
   - 视觉语言：真实的生活场景，自然光线，人物状态真实
   - 不要：商业广告感、过度美化、明星相关
   - 封面不带文字
   - 尺寸规格：严格为 900×383 像素（2.35:1 比例）
   - 封面设计参考：见 [references/cover-design-guide.md](references/cover-design-guide.md)

2. **生成正文配图**：
   - 根据文章主要场景，生成 2～4 张配图
   - 风格：纪实摄影感，或质朴的手绘插图风
   - 尺寸：900×500 像素（16:9 比例）
   - 每张配图应对应文中一个具体场景或情绪节点

3. **在 Markdown 中插入图片引用**：
   - 封面图：放在文章标题下方
   - 正文配图：放在对应段落后
   - 外部 URL 格式：`![图片描述](https://外部图片URL)`

**步骤4：排版优化**

真实故事计划专属排版风格：

1. **整体基调**：
   - 克制、沉稳、留白充足
   - 不用醒目配色，主色调为深灰/黑色
   - 文字密度不过高，段落间有呼吸感

2. **结构设计**：
   - 不使用大量小标题分割（非干货文，避免碎片化）
   - 若需分段，用**数字序号**（一、二、三）或空行分隔
   - 金句用引用样式（`>` 块）突出

3. **细节处理**：
   - 对话文字适当缩进或引用
   - 重点信息可加粗，但不滥用
   - 不使用任何分隔线（`---` / `———`）

4. **结尾互动**：
   - 文章结尾加一段轻柔的引导语，不强迫互动
   - 示例：「如果你也有一个想被记录的故事，欢迎留言。」

5. **隐私声明**：
   - 在文章最末尾、互动引导语之后，加一行灰色小字声明
   - 内容：「（出于隐私保护，文中人物为化名）」或「（为保护隐私，文中人名均为化名）」
   - **Markdown 格式**：使用引用语法 `>` 包裹，独立一行，例如：
     ```
     > （出于隐私保护，文中人物为化名）
     ```
   - **样式要求**：与口述引言一致的浅灰色小号字体（generate_html.py 已内置对应样式）

**排版参考**：见 [references/formatting-best-practices.md](references/formatting-best-practices.md)

1. **内容检查**：

   - 封面图尺寸是否为 900×383 像素？

## 注意事项
- **不消费苦难**：故事中的困境是用来理解的，不是用来感叹或猎奇的
- **不替主人公下结论**：叙述者呈现，读者感受，不需要代替主人公总结人生
- **第一人称不等于自我宣泄**：克制是力量，避免情绪外泄
- **细节是灵魂**：「一个外卖员骑着车」不如「他骑着那辆前轮有点晃的电动车」
- 排版不使用任何分隔线
- 图片标签格式：外部 URL 用 `![描述](https://URL)`，本地图片用 `![描述](./images/名.png)`
- 充分利用语言理解能力，优先以叙事能力驱动，减少套路

## 使用示例

**示例1：职场主题**
- 用户输入：「我想写一个在工厂流水线工作的年轻人的故事」
- 执行：智能体以第一人称叙事，描绘流水线工人的一个具体工作日，抓住细节，呈现内心
- 关键处理：避免「励志」叙事，聚焦普通日子里的疲倦、小快乐、无解的困惑

**示例2：情感主题**
- 用户输入：「我想写一个离婚单亲妈妈的故事」
- 执行：从某一个具体的生活瞬间切入（比如独自签完离婚协议当天），展开一段真实的生活状态
- 关键处理：不把她处理成「坚强独立女性」符号，展现复杂的内心和选择

**示例3：边缘群体**
- 用户输入：「我想写老年痴呆患者的家属故事」
- 执行：以照料者（子女/配偶）第一人称叙述，记录照料日常中的某个时刻，以小见大
- 关键处理：不煽情，用细节传递爱与疲惫共存的真实状态
