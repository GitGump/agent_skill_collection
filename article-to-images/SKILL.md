## 工作流程

### 第一步：获取文章内容

确认以下信息：
1. **文章标题**（必须）— 用于封面和文件夹命名
2. **文章正文**（必须）— 支持纯文本或 Markdown 格式
3. **来源/作者**（必须）— 显示在封面下方，**默认：好人古德曼**
4. **副标题/导语**（必须）— AI 根据文章核心内容生成一句吸引点击的导语，显示在封面标题下方
5. **指定主题**（可选）— 不指定时自动匹配，或 AI 自定义主题

文章内容来源可能是：
- 用户直接粘贴的文本（支持 Markdown）
- 需要从 URL 抓取的文章
- 工作区中的 .txt / .md 文件
- 用户指定路径的文件

**副标题生成规则**：
- 提炼文章最核心的冲突/悬念/情绪
- 控制在 30 字以内
- 避免剧透，但要足够勾起好奇心
- 示例：「曼彻斯特的敲门声，四十四年的冷战，和那些凌晨两点醒来的孤独」

### 第二步：主题匹配

#### 策略 A：内置主题自动匹配（默认）

根据文章关键词自动匹配最佳主题风格：

| 关键词 | 匹配主题 | 视觉特征 |
|--------|---------|---------|
| 科技/AI/编程/互联网 | 📘 科技蓝 | 蓝白底+蓝色标题+青绿点缀 |
| 商业/财经/股票/投资 | 💼 商务灰 | 浅灰底+深灰标题+金色装饰 |
| 文学/小说/故事/非虚构 | 📗 文艺绿 | 豆绿底+绿色标题+浅绿背景 |
| 生活/旅行/美食/日常 | 🍊 暖橙 | 暖白底+橙色标题+暖色点缀 |
| 设计/极简/哲学/观点 | ⬛ 极简黑白 | 白底+黑标题+灰色线条 |
| 新闻/时事/社会/热点 | 📕 杂志红 | 白底+红色标题+粉红背景 |
| 历史/复古/怀旧/经典 | 🏺 复古金 | 米色底+棕色标题+金色装饰 |
| 自然/环保/清新/风景 | 💙 清新蓝 | 浅蓝底+天蓝标题+蓝色点缀 |

**混搭规则**：如果文章跨多个领域，优先使用关键词命中次数最多的主题。

#### 策略 B：AI 自定义主题（v3.0 新增）

当内置主题效果不佳时，AI 可以自定义主题配色：

1. **选择主色**：根据文章气质选择一个十六进制主色（如 `#455A64` 曼彻斯特灰蓝）
2. **命名主题**：给主题一个直观的名字（如「曼彻斯特灰蓝」）
3. **选择背景类型**：
   - `light`（默认）：近白底色，极淡主色底
   - `warm`：暖白底色，适合温暖/怀旧类文章
   - `cool`：冷白底色，适合清冷/忧郁类文章
   - `dark`：深色背景，适合深沉/暗黑类文章
4. **脚本自动推导**：从主色自动计算标题色、强调色、背景色、文字色、装饰色等 14 个字段

**自定义主题设计规范**：
- 强调色取色环偏移 ~30° 的邻近色，和谐不冲突
- 文字与背景对比度满足 WCAG AA 标准
- 所有衍生色保持色相一致，仅调整明度/饱和度
- 确保封面背景与标题白字对比度足够

**使用示例**：
```bash
python {skill_dir}/scripts/generate_article_images.py \
  --title "The Smiths | 混蛋 Morrissey 和他的完美乐队" \
  --input article.txt \
  --subtitle "曼彻斯特的敲门声，四十四年的冷战" \
  --custom-primary-color "#455A64" \
  --custom-theme-name "曼彻斯特灰蓝" \
  --bg-type cool
```

### 第三步：Markdown 解析

如果文章是 Markdown 格式，自动解析以下元素：

| Markdown 语法 | 解析结果 | 渲染样式 |
|--------------|---------|---------|
| `# 标题` | H1 主标题 | 封面页大标题 |
| `## 标题` | H2 章节标题 | **加粗+下划线+主题色** |
| `### 标题` | H3 小节标题 | **加粗+下划线+主题色** |
| `**重点**` | 重点段落 | **高亮背景+加粗+主题色** |
| `> 引用` | 引用块 | 左边框+主题色斜体 |
| `- 列表` | 无序列表 | 主题色圆点 |
| `` `代码` `` | 代码块 | 浅色背景+等宽字体 |
| `---` | 分割线 | 主题色细线 |

### 第四步：执行图片生成

调用 `scripts/generate_article_images.py` 脚本生成图片：

```bash
python {skill_dir}/scripts/generate_article_images.py \
  --title "文章标题" \
  --text "文章正文内容" \
  --subtitle "副标题"
```

参数说明：
- `--title, -t`：文章标题（必填）
- `--text`：直接传入文章正文
- `--input, -i`：从文件读取文章（与 --text 二选一）
- `--output, -o`：输出目录路径（可选，默认保存到 `E:\codefile\trae\mp_article\文生图\整篇转换\{标题-时间戳-V1}`）
- `--source, -s`：作者/来源信息（**默认：好人古德曼**）
- `--subtitle`：副标题/导语（**必填** — AI 应生成）
- `--theme`：强制指定内置主题（可选，不填则自动匹配）
- `--custom-primary-color`：自定义主题主色十六进制值（如 `#8B4513`）
- `--custom-theme-name`：自定义主题显示名称
- `--bg-type`：自定义主题背景类型：`light`/`warm`/`cool`/`dark`

**主题列表**：tech / business / literature / lifestyle / minimal / magazine / retro / fresh

```bash
# 查看所有可用主题
python {skill_dir}/scripts/generate_article_images.py --list-themes

# 强制指定主题
python {skill_dir}/scripts/generate_article_images.py \
  --title "AI时代的思考" \
  --text "文章内容..." \
  --theme tech
```

**重要**：Windows PowerShell 下传递长文本时，优先使用 `--input` 参数将文章先保存到临时 .txt 文件，避免命令行参数长度限制。

### 第五步：生成公众号文案（copywriting.md）

AI 根据文章内容，按照以下结构生成 300 字以内的公众号文案，保存到输出目录下的 `copywriting.md`：

**文案结构**：

| 序号 | 模块 | 要求 |
|------|------|------|
| 1 | 吸引力标题 | 不超过 20 个字（含标点和 emoji） |
| 2 | 核心观点段 | 2-3 句，场景切入，从读者痛点或好奇心出发 |
| 3 | 要点列表 | 3-5 个，使用 `▸` 标记，长短参差 |
| 4 | 关键数据/金句 | 1-2 句，引用原文最具冲击力的数据或观点 |
| 5 | 立场声明 | 如适用，`⚠️` 标注利益相关 |
| 6 | 话题标签 | 5-8 个，格式 `#标签1 #标签2 ...` |

**文案风格**：遵循 `references/copywriting_guide.md` 的去 AI 味规范——口语化断句、个人视角切入、要点长短不一、有态度有判断。

**示例**（完整示范见 `references/copywriting.md`）：

```markdown
# 🎨 文字变身杂志，一键出图

你有没有这种感觉——写了三小时的好文章，发到朋友圈就是一行灰色链接，根本没人点？

说白了，问题不在内容，在呈现。同样的文字，杂志排版和记事本排版的阅读体验差了十条街。

- ▸ 长文切 9 张图，每张一屏，拇指一划就看完——跟刷小红书一样顺
- ▸ 主题色自动匹配，文艺文用豆绿，商业文用灰金，不用你纠结配色
- ▸ 每篇 300 字文案 2-3 张图，不啰嗦，不浪费读者的拇指

> 好内容值得好排版。一张灰色链接，不如九张有呼吸感的图。

⚠️ 作者即工具开发者，但工具免费、文案纯手打，建议自行体验后判断。

#内容创作 #图文排版 #文章转图 #小红书图文 #公众号排版 #写作工具 #自媒体运营
```

**保存路径**：与切片图片同一输出目录下的 `copywriting.md` 文件。

### 第六步：HTML 样式预览

脚本会自动在同一输出目录下生成 `preview.html` 文件，包含：
- 完整的文章排版预览
- 与切片图一致的配色和样式
- 可在浏览器中直接打开预览排版效果

### 第七步：交付结果

1. 告知用户生成了多少张图片
2. 告知匹配的主题风格
3. 告知图片保存的文件夹路径
4. 告知 HTML 预览文件路径
5. 告知文案文件路径（copywriting.md）
6. 使用 `preview_url` 展示 HTML 预览效果
7. 使用 `deliver_attachments` 将所有图片和 copywriting.md 作为附件交付

## 设计规范

### 通用规范

| 元素 | 规格 |
|------|------|
| 图片尺寸 | 1080 × 1440 px（3:4 竖版） |
| 正文字号 | 30px 微软雅黑 |
| 标题字号 | 40px 微软雅黑粗体（H2）/ 34px（H3） |
| 封面标题 | 44-64px 微软雅黑粗体（自动缩放） |
| 正文行距 | 1.5 倍字号 |
| 段落间距 | 45px（1.5 行文字高度） |
| 段标题下方间距 | 45px（1 行文字高度） |
| 左右边距 | 60px |
| 上边距 | 60px |
| 下边距 | 80px |
| 最大页数 | 18 张 |

### 封面页设计（首张切片）

- **上方 2/3 区域**：主题色背景块
- **标题**：白色大字居中（44-64px，超长自动缩小）
- **副标题**：主题色浅色文字
- **下方 1/3 区域**：白色底
  - 来源/作者信息
  - **无主题名称标签**（v2.1 去除）
  - **无标题区域装饰横线**（v2.2 去除）

### 正文页设计

- **段落标题**：加粗 + 下划线 + 主题色（标题色），**下方 45px 间隔**（v2.2）
- **重点内容**（Markdown **加粗**）：高亮背景 + 加粗 + 主题色
- **引用块**：左边框 + 主题色文字 + 浅色背景
- **列表**：主题色圆点 + 标准缩进
- **代码块**：浅色背景 + 等宽字体
- **页码**：右下角，"当前/总数" 格式
- **无顶部装饰线**（v2.2 去除）
- **底部装饰线**：页码正上方（v2.2 下移）
- **无主题标签**（v2.1 去除左下角主题名称）

## 注意事项

- 脚本自动处理 Windows 控制台编码问题
- Markdown 格式文章会先解析为结构化内容块再分页
- 文章过长时自动分多页，基于像素级精确计算
- **最大页数限制为 18 张**（仅包含封面和正文页），超出时会截断内容
- 分页优先在段落边界处断开
- 标题中的特殊字符会自动替换为下划线以保证文件名合法
- 字体自动回退：微软雅黑 → 黑体 → 系统默认
- 默认输出路径：`E:\codefile\trae\mp_article\文生图\整篇转换\{标题-时间戳-V1}`
- 每次生成同时输出 `preview.html` 样式预览文件
- 所有切片**不显示主题名称标签**（v2.1 去除）
- 段落间距 45px，确保视觉清晰分隔
- 封面页**无标题区域装饰横线**（v2.2 去除）
- 正文页**无顶部装饰线**，底部装饰线位于页码正上方（v2.2）
- 段标题下方 45px 间隔（v2.2）
- **副标题/导语必填**（v3.0）— AI 为每篇文章生成吸引点击的导语
- **来源/作者必填**（v3.0）— 默认「好人古德曼」
- **支持 AI 自定义主题**（v3.0）— 传入主色自动推导完整配色方案
- **新增 copywriting.md 产物**（v3.1）— AI 为每篇文章生成 300 字以内配套公众号文案
