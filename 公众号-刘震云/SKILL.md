# 微信公众号文案写作助手（刘震云风格）

## 任务目标
- 本Skill用于：快速生成高质量的微信公众号文案，从内容创作、配图生成再到自动排版的一站式服务
- 能力包含：
  1. 检索近期热点和热门选题
  2. 以刘震云的写作风格生成2000字左右的公众号文案（2000~5000字）
  3. 自动生成正文配图和封面图
  4. 自动完成文章排版优化（适合公众号展示）
  5. 生成可直接复制到公众号的HTML文件（保留完整排版）
  6. 提供发布到公众号的操作指引
- 触发条件：用户提供创作主题/想法，说"帮我写一篇公众号文章"或类似表达

## 前置准备
- 依赖说明：scripts脚本所需的依赖包及版本
  ```
  markdown>=3.4.0
  beautifulsoup4>=4.12.0
  ```

## 操作步骤

### 标准流程

**步骤1：热点选题**
基于用户提供的主题，智能体将：

1. **检索近期热点**：
   - 搜索与主题相关的近期热点事件、热门话题
   - 筛选1-3个最具传播潜力的切入点
   - 分析热点与用户主题的关联性，确定最佳切入点

**热点选题参考**：见 [references/hot-topic-search.md](references/hot-topic-search.md)

**步骤2：文章创作**
以刘震云的写作风格创作文章：

**刘震云写作风格特点**：
- **语言风格**：朴实直白、冷幽默、口语化强，像农民说话。偶尔冷不丁来一个咸的玩笑，幽默但不过分，初读觉得好玩，读完回味又觉得意味深远。
- **表达方式**：大量使用反讽、自嘲、克制夸张、朴素比喻。偶尔穿插典故原文片段或金句，进行恰当的引用与解构，用来辅助行文推进以及观点佐证。
- **结构特点**：开篇从小事切入、中段绕来绕去越绕越深、结尾恍然大悟或意味深长
- **句式特点**：短句与长句相结合，节奏感好，可读性强。回环往复、重复强调、轻描淡写。不追求全部都是短句，长短交错，读起来才有呼吸感。
- **内容特点**：用故事讲道理、小中见大、接地气。穿插经典典故或金句，先引用原文，再解构分析，让文章更有文化厚度和思想深度。

1. **生成文章结构**：
   - 拟定大纲：3-5个主要章节，每个章节围绕一个核心观点反复讲
   - 设计开头：从一件小事/一个人/一句话切入，小中见大
   - 制造回环：同一个观点换个说法再说，越说越深
   - 规划金句插入点：在每个章节的关键转折处，预先规划好1-2个典故或金句的引用位置

2. **撰写正文内容**：
   - **字数控制**：2000字左右（2000~5000字范围内）
   - **句式节奏**：短句与长句相结合，营造呼吸感。短句有力道，长句有韵味。像说话，但不是每句都短——那样读起来像打电报。
   - **幽默风格**：加入更多幽默元素，不是刻意搞笑，而是冷不丁来一个咸的玩笑或意外的比喻。比如写人生艰难，可以突然来一句"比上班还难"；写人际关系，可以说"有些人说话的艺术在于，你听完觉得他说了一整天，其实他就说了三个字'随便吧'"。
   - **金句与典故运用**（重要创新）：
     - 在正文中合适处穿插典故原文片段或经典金句（如古诗文、名言警句、小说精彩段落等）
     - 引用格式：先引用原文，保持原汁原味
     - 解构分析：对引用内容进行拆解、调侃、升华，让老道理说出新意思
     - 用法示例："都说'人生如戏'，但刘震云老师说了，戏还有谢幕的时候，人生这出戏，有些人连领盒饭的机会都没有。"
     - 位置选择：引言通常出现在段落开头（作为论点引入）或段落结尾（作为升华收束）
     - 频次控制：每800-1000字至少穿插一处金句引用，全文不超过3-4处，避免堆砌
   - **保持刘震云的语言风格**：朴实、冷幽默、回环往复
   - 用反讽、自嘲、克制夸张、朴素比喻等修辞手法
   - 回环往复，同一件事绕来绕去，越绕越深
   - 穿插热点话题和实际案例，增强可读性。必须使用最新热点案例（优先使用最近1-2个月内的热门AI工具或事件），避免使用过时案例。**科技类文章尤其要注意时效性，如使用Clawdbot、Grok-3、DeepSeek等最新AI工具作为案例，而非ChatGPT等早期工具。案例需包含具体时间点、数据支撑和具体场景，如"上周发布的Clawdbot V3版本"或"某机构测试显示转化率提升40%"。
   - 贴近生活，像拉家常
   - 设计结尾：要么恍然大悟，要么意味深长，不说透

3. **优化润色**：
   - 检查段落逻辑是否连贯，是否在"绕"中越说越深
   - 确保观点清晰、论证有力但不直接说透
   - 调整语言表达，符合刘震云风格
   - 控制节奏，重复是艺术不是啰嗦
   - 检查金句引用是否恰当，是否与上下文自然融合
   - 检查句式是否有长短变化，避免全部短句造成的急促感

4. **生成爆款标题**（文章完成后）：
   - 基于完成的文章内容，提炼核心亮点和关键信息
   - 确定标题：**朴实有味的标题**，符合刘震云的标题风格，不花哨但有深意
   - 标题生成参考：见 [references/viral-title-best-practices.md](references/viral-title-best-practices.md)
     - **核心内容**：爆款标题特质、设计要素、不同赛道风格（科技/情感/职场/教育/理财/健康/美食/旅游/时尚）
     - **生成方法**：5步法生成流程、6种标题模板（数字型/疑问句/对比句/金句型/感叹句/痛点型）
     - **高级技巧**：多维度组合、情绪引导、时效性、社交属性
     - **实战案例**：失败案例分析和AB测试对比实例
   - 生成多个标题备选（3-5个），选择最优标题
   - 确保标题与文章内容高度匹配，避免标题党

**步骤3：配图生成与引用插入**
智能体将为文章生成配图并在Markdown中插入引用：

1. **生成封面图**：
   - 根据文章主题和风格，生成一张高质量的封面图
   - 使用步骤2生成的爆款标题作为封面生图的风格主题参考，美观协调
   - 封面不要带文字
   - 尺寸规格：严格为900×383像素（2.35:1比例）
   - 设计要求：遵循爆款封面设计原则，确保视觉冲击力、信息传达效率、品牌辨识度
     - 封面设计参考：见 [references/cover-design-best-practices.md](references/cover-design-best-practices.md)
       - **核心内容**：爆款封面特质、设计要素（构图/色彩/文字/元素）、不同赛道风格详解
       - **更多风格**：插画风格、真人写实风格、极简风格、3DCg风格、油画风格、赛博朋克风格
       - **实例库**：8个不同赛道的封面设计实例（科技/情感/职场/教育/理财/健康/美食/旅游）
       - **工具资源**：设计工具推荐、图片资源、配色工具
   - 构图：简洁大气，突出核心信息，留白充足
   - 色彩：符合赛道风格

2. **生成正文配图**：
   - 根据每个主要章节的内容，生成2-4张正文配图
   - 风格要求：与文章风格匹配、图文相关、美观协调
   - 尺寸建议：900×500像素（16:9比例）

3. **在Markdown中插入图片引用**（重要）：
   - **必须**在生成图片后，立即在Markdown文章中插入图片引用标签
   - **推荐使用外部URL**：
     - Markdown中使用：`![图片描述](https://外部图片URL)`
     - 获取图片URL：智能体生成图片后，通常提供图片链接
     - 如果没有提供URL，使用占位符：`![图片描述](https://待替换的图片URL)`
   - **图片插入位置**：
     - 封面图：在文章开头标题下方插入
     - 正文配图：在每个小标题后的合适位置插入
   - **图片命名规则**（便于管理）：
     - 封面图：`![封面图](https://xxx.png)`
     - 正文配图：`![配图1-章节主题](https://xxx.png)`、`![配图2-章节主题](https://xxx.png)`

**步骤4：自动排版优化**
智能体将为文章进行专业的公众号排版优化：

1. **结构优化**：
   - 添加吸引眼球的标题（加大加粗）
   - 设置简短摘要或引语（用引用样式）
   - 合理划分段落（每段3-5句话，避免过长）
   - 使用小标题分割内容（三级或四级标题）
   - **注意：不使用分隔线**

2. **视觉优化**：
   - 重点内容加粗（关键词、核心观点）
   - 金句和重要数据用引用样式突出
   - 关键数字使用特殊标记（如>100万）
   - 列表化要点（使用有序或无序列表）
   - 长段落拆分为短句，提升可读性
   - **典故引用样式**：引用原文时使用引用块突出显示，解构分析时用正常段落

3. **配图位置优化**：
   - 检查所有图片引用是否正确插入
   - 确保图片位置合理，与内容相关
   - 图片前后留有适当间距

4. **互动设计**：
   - 文章结尾添加引导互动的话语
   - 设置问题引导读者留言
   - 添加"点赞、在看、转发"引导

5. **排版样式示例**：
   ```
   [大标题：加粗、24号字]

   [摘要/引语：引用样式、16号字、灰色]

   ![封面图](https://外部图片URL)

   [小标题1：加粗、18号字]

   [正文段落：16号字、行间距1.75]
   **重点内容加粗**，普通内容保持原样。
   
   > 典故引用：保持原文原汁原味
   [对典故的解构分析，用朴实直白的语言]

   ![配图1-章节主题](https://外部图片URL)

   - 要点1：使用列表
   - 要点2：清晰明了
   - 要点3：易于阅读

   [小标题2：加粗、18号字]

   [正文段落：16号字、行间距1.75]

   ![配图2-章节主题](https://外部图片URL)

   [结尾：引导互动]
   您怎么看？欢迎在评论区留言讨论~
   点赞和"在看"，是对我最好的支持👇
   ```

**图片嵌入策略**：
- **推荐方式：使用外部URL**
  - Markdown中使用：`![描述](https://s.coze.cn/image/xxx/)`
  - HTML文件更小，加载更快
  - 图片稳定显示，不受本地环境限制
  - 复制到公众号后图片自动加载

- **备选方式：本地图片转base64**
  - Markdown中使用：`![描述](./images/filename.jpg)`
  - 运行时指定images目录：`python generate_html.py article.md output.html images`
  - 优点：图片内嵌，无需单独上传
  - 缺点：HTML文件可能过大，浏览器兼容性问题

**排版优化参考**：见 [references/formatting-best-practices.md](references/formatting-best-practices.md)（公众号排版的专业技巧和样式指南）

**步骤5：生成HTML文件**
智能体将调用脚本生成可直接复制的HTML文件：

1. **脚本调用**：
   - 调用 `scripts/generate_html.py` 处理Markdown格式文章

   - 生成包含完整内联样式的HTML文件
   - HTML文件可在浏览器中直接打开

2. **样式优化**：
   - 所有样式内联，确保复制到公众号后保持效果
   - 标题、段落、引用、列表等元素都有对应样式
   - 图片占位符更友好，明确标注尺寸要求

3. **图片处理说明**：
   - **情况1：Markdown中使用外部URL**（推荐）
     - Markdown标签：`![封面图](https://xxx.png)` 或 `![配图1](https://xxx.png)`
     - HTML会直接使用URL，无需额外处理
     - 复制到公众号后，图片会自动加载
     - HTML文件更小，加载更快
     - 图片稳定显示

   - **情况2：有本地图片文件**
     - Markdown标签：`![封面图](./images/封面图.png)` 或 `![配图1](./images/配图1.png)`
     - 智能体将图片保存到 `./images` 目录
     - 调用脚本：`python generate_html.py article.md article.html ./images`
     - 图片会自动转换为base64并嵌入到HTML中
     - 复制到公众号后，图片自动显示
     - 注意：HTML文件可能过大

   - **情况3：没有图片文件或URL**（不推荐）
     - 使用占位符：`📌 [图片描述]`
     - 明确标注建议尺寸：
       - 封面图：900×383像素（2.35:1比例）
       - 正文配图：900×500像素（16:9比例）
     - 提示用户上传配图或提供图片URL

## 使用示例
根据实际功能提供2-3个典型使用场景，每个示例包含：

**示例1：职场主题文章创作**
- 功能说明：创作一篇关于职场沟通的公众号文章
- 执行方式：智能体创作（刘震云风格）+ 脚本生成HTML
- 关键参数或指导要点：
  - 主题：职场沟通
  - 风格：刘震云（朴实直白、冷幽默、回环往复、短句与长句结合）
  - **字数：2000字左右（2000~5000字）**
  - **幽默元素：冷不丁来一个咸的玩笑**
  - **金句引用：穿插典故原文片段或金句，进行解构分析**
  - 标题：3-5个备选，选择最优
  - 封面：900×383像素
  - 配图：2-4张，900×500像素
  - **图片引用：在Markdown中插入外部URL图片标签**
- 简单示例代码或命令：
  ```bash
  # 推荐方式：使用外部URL
  python scripts/generate_html.py article.md article.html

  # 备选方式：使用本地图片
  python scripts/generate_html.py article.md article.html ./images
  ```
