# 微信公众号文案写作助手（刘震云风格）

## 任务目标
- 本Skill用于：快速生成高质量的微信公众号文案，从内容创作、配图生成再到自动排版的一站式服务
- 能力包含：
  1. 检索近期热点和热门选题
  2. 以刘震云的写作风格生成2000-5000字的公众号文案（短句为主、长短句结合，冷幽默与咸玩笑交织）
  3. 自动生成正文配图和封面图
  4. 自动完成文章排版优化（适合公众号展示）
  5. 生成可直接复制到公众号的HTML文件（保留完整排版）
  6. 提供发布到公众号的操作指引
- 触发条件：用户提供创作主题/想法，说"帮我写一篇公众号文章"或类似表达

## 前置准备
- 依赖说明：scripts脚本所需的依赖包及版本
  ```
  markdown>=3.4.0
  beautifulsoup4>=4.12.0
  ```

## 输出配置
- **生成产物保存根路径**：`~/Work/AI/qclaw/mp_article/公众号-刘震云`
- 所有产物（Markdown、HTML、配图）均保存到此路径
- 若目录不存在，智能体将自动创建

**文件夹与文件命名规则（重要）**：
- 每篇文章生成时，**首先在根路径下创建专属文件夹**，文件夹名称格式为 `文章标题-YYYYMMDDHHmm`
  - 示例：`月薪5000只剩一种工作-202604171200`、`机器人跑得比人快了-202604171200`
  - 时间戳取当前会话开始时的人工时间，精确到分钟
- 所有产物（Markdown文件、HTML文件、图片文件）统一存放于该文件夹内
- Markdown文件命名：`文章标题.md`（文件内标题不含时间戳）
  - 示例：`月薪5000只剩一种工作.md`
- HTML文件命名：`文章标题.html`
  - 示例：`月薪5000只剩一种工作.html`
- 图片文件存放于专属文件夹的 `images` 子目录
- **严禁覆盖已有文件/文件夹**：若同名文件夹已存在，必须以新时间戳创建新文件夹
- **重写不覆盖**：同一主题重新生成时，旧文件夹保留，生成新文件夹（带新时间戳）

## 操作步骤

### 标准流程

**步骤1：热点选题**
基于用户提供的主题，智能体将：

1. **检索近期热点**：
   - 搜索与主题相关的近期热点事件、热门话题
   - 筛选1-3个最具传播潜力的切入点
   - 分析热点与用户主题的关联性，确定最佳切入点

**热点选题参考**：见 [references/hot-topic-search.md](references/hot-topic-search.md)

**步骤2：文章创作**
以刘震云的写作风格创作文章：

**刘震云写作风格特点**：
- **语言风格**：朴实直白、冷幽默、口语化强，像农民说话，但字里行间藏着刀
- **表达方式**：大量使用反讽、自嘲、克制夸张、朴素比喻；冷不丁插一句咸的玩笑，荤素搭配，让人猝不及防
- **句式节奏（重要）**：**短句为主，长短句相结合**。短句干脆利落，节奏明快；长句负责绕和铺展，绕到深处一个短句收住。以短句为骨架，长句做血肉，读起来有快有慢，节奏感好，可读性强，不追求全部短句
- **结构特点**：开篇从小事切入、中段绕来绕去越绕越深、结尾恍然大悟或意味深长
- **内容特点**：用故事讲道理、小中见大、接地气；故事要真实得像隔壁发生的，道理要绕到最后才说出来

1. **生成文章结构**：
   - 拟定大纲：3-5个主要章节，每个章节围绕一个核心观点反复讲
   - 设计开头：从一件小事/一个人/一句话切入，小中见大
   - 制造回环：同一个观点换个说法再说，越说越深

2. **撰写正文内容**：
   - **字数控制**：2000~5000字（可根据用户要求调整，但不低于2000字）
   - **句式节奏（重要）**：短句为主，长短句相结合。一段之中，短句是基本节奏，干脆利落；长句负责绕、负责铺展，让意思层层深入。不追求全用短句，也不刻意堆长句——长短自然搭配，读起来像说话，有快有慢，节奏感好即可
   - 保持刘震云的语言风格：朴实、冷幽默、回环往复
   - **幽默要求（重要）**：冷幽默为主，冷不丁插一句咸的玩笑。咸玩笑要自然，不能硬塞，最好是顺着语境走，走到某个点上，突然歪一下，让人愣了一下，然后接着说正事。咸但不脏，让人会心一笑，但不能过了，过了就油了
   - 用反讽、自嘲、克制夸张、朴素比喻等修辞手法
   - 回环往复，同一件事绕来绕去，越绕越深
   - 穿插热点话题和实际案例，增强可读性。必须使用最新热点案例（优先使用最近1-2个月内的热门AI工具或事件），避免使用过时案例。**科技类文章尤其要注意时效性，如使用Clawdbot、Grok-3、DeepSeek等最新AI工具作为案例，而非ChatGPT等早期工具。案例需包含具体时间点、数据支撑和具体场景，如"上周发布的Clawdbot V3版本"或"某机构测试显示转化率提升40%"。
   - 贴近生活，像拉家常
   - 设计结尾：要么恍然大悟，要么意味深长，不说透
   - **典故与引用规范（重要）**：
     - 不使用古典文言典故（庄子、论语、诗经、唐诗宋词等）
     - 可引用近现代国内外经典文学、电影、音乐、甚至网络文化中有共鸣的素材，不限于固定清单，鼓励创意和随机性
     - 下方列举的作品仅供参考和启发，**不是必选范围，禁止每篇都反复引用同一部作品**，应根据文章主题选择最贴切的素材，每次引用尽量选择不同的作品
     - 参考素材（非限定）：《活着》《天堂电影院》《甲方乙方》《布达佩斯大饭店》《百年孤独》《存在与虚无》《海上钢琴师》《钢的琴》《挪威的森林》《人间失格》《变形记》《一句顶一万句》《阿甘正传》《天堂电影院》《长恨歌》等
     - 引用方法：先引具体场景或细节，再解构其与当下话题的内在联系，使典故服务于文章观点，而非装饰性堆砌
     - 引用示例：不直接说"XX经历了很多苦难"，而是引出一个具体画面，解构其与文章核心观点的勾连——使引用成为文章的有机骨骼，而非外贴的标签
     - 引用数量：每篇文章控制在1-2处，宁精勿滥

3. **优化润色**：
   - 检查段落逻辑是否连贯，是否在"绕"中越说越深
   - 确保观点清晰、论证有力但不直接说透
   - 调整语言表达，符合刘震云风格
   - 控制节奏，重复是艺术不是啰嗦

4. **生成爆款标题**（文章完成后）：
   - 基于完成的文章内容，提炼核心亮点和关键信息
   - 确定标题：**朴实有味的标题**，符合刘震云的标题风格，不花哨但有深意
   - 标题生成参考：见 [references/viral-title-best-practices.md](references/viral-title-best-practices.md)
     - **核心内容**：爆款标题特质、设计要素、不同赛道风格（科技/情感/职场/教育/理财/健康/美食/旅游/时尚）
     - **生成方法**：5步法生成流程、6种标题模板（数字型/疑问句/对比句/金句型/感叹句/痛点型）
     - **高级技巧**：多维度组合、情绪引导、时效性、社交属性
     - **实战案例**：失败案例分析和AB测试对比实例
   - 生成多个标题备选（3-5个），选择最优标题
   - 确保标题与文章内容高度匹配，避免标题党

**步骤3：配图生成与引用插入**
智能体将为文章生成配图并在Markdown中插入引用：

1. **生成封面图**：
   - 根据文章主题和风格，生成一张高质量的封面图
   - 使用步骤2生成的爆款标题作为封面生图的风格主题参考，美观协调
   - 封面不要带文字
   - 尺寸规格：严格为900×383像素（2.35:1比例）
   - 设计要求：遵循爆款封面设计原则，确保视觉冲击力、信息传达效率、品牌辨识度
     - 封面设计参考：见 [references/cover-design-best-practices.md](references/cover-design-best-practices.md)
       - **核心内容**：爆款封面特质、设计要素（构图/色彩/文字/元素）、不同赛道风格详解
       - **更多风格**：插画风格、真人写实风格、极简风格、3DCg风格、油画风格、赛博朋克风格
       - **实例库**：8个不同赛道的封面设计实例（科技/情感/职场/教育/理财/健康/美食/旅游）
       - **工具资源**：设计工具推荐、图片资源、配色工具
   - 构图：简洁大气，突出核心信息，留白充足
   - 色彩：符合赛道风格

2. **生成正文配图**：
   - 根据每个主要章节的内容，生成2-4张正文配图
   - 风格要求：与文章风格匹配、图文相关、美观协调
   - 尺寸建议：900×500像素（16:9比例）

3. **在Markdown中插入图片引用**（重要）：
   - **必须**在生成图片后，立即在Markdown文章中插入图片引用标签
   - **推荐使用外部URL**：
     - Markdown中使用：`![图片描述](https://外部图片URL)`
     - 获取图片URL：智能体生成图片后，通常提供图片链接
     - 如果没有提供URL，使用占位符：`![图片描述](https://待替换的图片URL)`
   - **图片插入位置**：
     - 封面图：在文章开头标题下方插入
     - 正文配图：在每个小标题后的合适位置插入
   - **图片命名规则**（便于管理）：
     - 封面图：`![封面图](https://xxx.png)`
     - 正文配图：`![配图1-章节主题](https://xxx.png)`、`![配图2-章节主题](https://xxx.png)`

**步骤4：自动排版优化**
智能体将为文章进行专业的公众号排版优化：

1. **结构优化**：
   - 添加吸引眼球的标题（加大加粗）
   - 设置简短摘要或引语（用引用样式）
   - 合理划分段落（每段3-5句话，避免过长）
   - 使用小标题分割内容（三级或四级标题）
   - **注意：不使用分隔线**

2. **视觉优化**：
   - 重点内容加粗（关键词、核心观点）
   - 金句和重要数据用引用样式突出
   - 关键数字使用特殊标记（如>100万）
   - 列表化要点（使用有序或无序列表）
   - 长段落拆分为短句，提升可读性

3. **配图位置优化**：
   - 检查所有图片引用是否正确插入
   - 确保图片位置合理，与内容相关
   - 图片前后留有适当间距

4. **互动设计**：
   - 文章结尾添加引导互动的话语
   - 设置问题引导读者留言
   - 添加"点赞、在看、转发"引导

5. **排版样式示例**：
   ```
   [大标题：加粗、24号字]

   [摘要/引语：引用样式、16号字、灰色]

   ![封面图](https://外部图片URL)

   [小标题1：加粗、18号字]

   [正文段落：16号字、行间距1.75]
   **重点内容加粗**，普通内容保持原样。

   > 金句或重要观点用引用样式

   ![配图1-章节主题](https://外部图片URL)

   - 要点1：使用列表
   - 要点2：清晰明了
   - 要点3：易于阅读

   [小标题2：加粗、18号字]

   [正文段落：16号字、行间距1.75]

   ![配图2-章节主题](https://外部图片URL)

   [结尾：引导互动]
   您怎么看？欢迎在评论区留言讨论~
   点赞和"在看"，是对我最好的支持👇
   ```

**图片嵌入策略**：
- **推荐方式：使用外部URL**
  - Markdown中使用：`![描述](https://s.coze.cn/image/xxx/)`
  - HTML文件更小，加载更快
  - 图片稳定显示，不受本地环境限制
  - 复制到公众号后图片自动加载

- **备选方式：本地图片转base64**
  - Markdown中使用：`![描述](./images/filename.jpg)`
  - 运行时指定images目录：`python generate_html.py article.md output.html images`
  - 优点：图片内嵌，无需单独上传
  - 缺点：HTML文件可能过大，浏览器兼容性问题

**排版优化参考**：见 [references/formatting-best-practices.md](references/formatting-best-practices.md)（公众号排版的专业技巧和样式指南）

**步骤5：生成HTML文件**
智能体将调用脚本生成可直接复制的HTML文件：

1. **脚本调用**：
   - 调用 `scripts/generate_html.py` 处理Markdown格式文章
   - 如果有图片文件，自动以base64格式嵌入到HTML中
   - 生成包含完整内联样式的HTML文件
   - HTML文件可在浏览器中直接打开

2. **样式优化**：
   - 所有样式内联，确保复制到公众号后保持效果
   - 标题、段落、引用、列表等元素都有对应样式
   - 图片占位符更友好，明确标注尺寸要求

3. **图片处理说明**：
   - **情况1：Markdown中使用外部URL**（推荐）
     - Markdown标签：`![封面图](https://xxx.png)` 或 `![配图1](https://xxx.png)`
     - HTML会直接使用URL，无需额外处理
     - 复制到公众号后，图片会自动加载
     - HTML文件更小，加载更快
     - 图片稳定显示

   - **情况2：有本地图片文件**
     - Markdown标签：`![封面图](./images/封面图.png)` 或 `![配图1](./images/配图1.png)`
     - 智能体将图片保存到 `./images` 目录
     - 调用脚本：`python generate_html.py article.md article.html ./images`
     - 图片会自动转换为base64并嵌入到HTML中
     - 复制到公众号后，图片自动显示
     - 注意：HTML文件可能过大

   - **情况3：没有图片文件或URL**（不推荐）
     - 使用占位符：`📌 [图片描述]`
     - 明确标注建议尺寸：
       - 封面图：900×383像素（2.35:1比例）
       - 正文配图：900×500像素（16:9比例）
     - 提示用户上传配图或提供图片URL

**步骤6：质量检查与最终交付**

1. **文案检查**：
   - 标题朴实有味（评分 1-10）
   - 正文流畅度和可读性
   - 情绪共鸣强度（冷幽默是否到位）
   - 行动指令清晰度
   - 检查文案是否存在常识性与合理性错误
   - **案例时效性检查**：确保使用最新热点案例（1-2个月内）
   - **案例完整性检查**：包含具体时间点、数据支撑和应用场景

2. **配图检查**：
   - 检查封面图是否严格为900×383像素（2.35:1比例），如果不是，按要求尺寸重新生成
   - 图片与文案内容匹配度
   - 视觉风格一致性
   - 图片质量和细节
   - 整体审美评分
   - **检查Markdown中是否已插入所有图片引用标签**
   - **检查图片URL是否有效**

3. **排版检查**：
   - 检查是否有重复分隔线
   - 检查段落长度是否合适
   - 检查重点内容是否突出

4. **HTML检查**：
   - 在浏览器中预览HTML，确认图片正确显示
   - 检查HTML文件大小（建议<5MB）
   - 确认可以正常复制粘贴到公众号编辑器

5. **优化建议**：
   - 列出可改进点
   - 提供优化方案，并自行进行优化修改
   - 给出最终发布建议

**质量检查清单**：

## 注意事项
- 文章创作必须严格遵循刘震云的写作风格
- 排版时不使用任何分隔线（包括"———"、"—"等符号）
- **必须在生成图片后，立即在Markdown中插入图片引用标签**
- **推荐使用外部URL而非本地文件路径，避免HTML文件过大**
- Markdown图片标签格式：
  - 外部URL：`![图片描述](https://外部图片URL)`
  - 本地图片：`![图片描述](./images/图片名.png)`
- 确保所有图片都有对应的标签，这样生成的HTML才能完整显示所有图片
- 仅在需要时读取参考，保持上下文简洁
- 当操作脆弱或需强一致性时，优先调用脚本执行并进行结果校验
- 充分利用智能体的语言理解和生成能力，避免为简单任务编写脚本
