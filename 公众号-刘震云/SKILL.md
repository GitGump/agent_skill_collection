# 微信公众号文案写作助手（刘震云风格）

## 任务目标
- 本Skill用于：快速生成高质量的微信公众号文案，从内容创作、事实筛查再到自动排版的一站式服务
- 能力包含：
  1. 检索近期热点和热门选题
  2. 以刘震云的写作风格生成2000-5000字的公众号文案（短长句结合节奏感强，冷幽默与咸玩笑交织）
  3. 对所有事实性内容进行准确性验证（时间、数据、引用、观点），确保信息真实可靠
  4. 自动完成文章排版优化（适合公众号展示）
  5. 生成可直接复制到公众号的HTML文件（保留完整排版）
  6. 提供发布到公众号的操作指引
- 触发条件：用户提供创作主题/想法，说"帮我写一篇公众号文章"或类似表达

## 输出配置
- **生成产物保存根路径**：`E:\codefile\trae\mp_article\公众号-刘震云`
- 所有产物（Markdown、HTML）均保存到此路径
- 若目录不存在，智能体将自动创建

**文件夹与文件命名规则（重要）**：
- 每篇文章生成时，**首先在根路径下创建专属文件夹**，文件夹名称格式为 `文章标题-YYYYMMDDHHmm`
  - 示例：`月薪5000只剩一种工作-202604171200`、`机器人跑得比人快了-202604171200`
  - 时间戳取当前会话开始时的人工时间，精确到分钟
- 所有产物（Markdown文件、HTML文件）统一存放于该文件夹内
- Markdown文件命名：`文章标题.md`（文件内标题不含时间戳）
  - 示例：`月薪5000只剩一种工作.md`
- HTML文件命名：`文章标题.html`
  - 示例：`月薪5000只剩一种工作.html`
- **严禁覆盖已有文件/文件夹**：若同名文件夹已存在，必须以新时间戳创建新文件夹
- **重写不覆盖**：同一主题重新生成时，旧文件夹保留，生成新文件夹（带新时间戳）

## 操作步骤

### 标准流程

**步骤1：热点选题**
基于用户提供的主题，智能体将：

1. **检索近期热点**：
   - 搜索与主题相关的近期热点事件、热门话题
   - 筛选1-3个最具传播潜力的切入点
   - 分析热点与用户主题的关联性，确定最佳切入点

**热点选题参考**：见 [references/hot-topic-search.md](references/hot-topic-search.md)

**步骤2：文章创作**
以刘震云的写作风格创作文章：

**刘震云写作风格特点**：
- **语言风格**：朴实直白、冷幽默、口语化强，像农民说话，但字里行间藏着刀
- **表达方式**：大量使用反讽、自嘲、克制夸张、朴素比喻；冷不丁插一句咸的玩笑，荤素搭配，让人猝不及防
- **句式节奏（重要）**：**短句与长句交替使用**。短句干脆利落，像刀切；长句一口气绕到底，绕到人晕了，突然一个短句收住。节奏感要好，读起来像走路，有时快有时慢，但不会让人停下来
- **结构特点**：开篇从小事切入、中段绕来绕去越绕越深、结尾恍然大悟或意味深长
- **内容特点**：用故事讲道理、小中见大、接地气；故事要真实得像隔壁发生的，道理要绕到最后才说出来

1. **生成文章结构**：
   - 拟定大纲：3-5个主要章节，每个章节围绕一个核心观点反复讲
   - 设计开头：从一件小事/一个人/一句话切入，小中见大
   - 制造回环：同一个观点换个说法再说，越说越深

2. **撰写正文内容**：
   - **字数控制**：2000~5000字（可根据用户要求调整，但不低于2000字）
   - **句式节奏（重要）**：短句与长句交替使用。一段之中，长句负责绕，短句负责切。长句一口气写到底，中间可以有逗号、顿号，但不要一口气用到底——每隔三四句插一个短句，让读者喘口气，也让节奏不拖沓。读起来像说话，有快有慢，有急有缓
   - 保持刘震云的语言风格：朴实、冷幽默、回环往复
   - **幽默要求（重要）**：冷幽默为主，冷不丁插一句咸的玩笑。咸玩笑要自然，不能硬塞，最好是顺着语境走，走到某个点上，突然歪一下，让人愣了一下，然后接着说正事。咸但不脏，让人会心一笑，但不能过了，过了就油了
   - 用反讽、自嘲、克制夸张、朴素比喻等修辞手法
   - 回环往复，同一件事绕来绕去，越绕越深
   - 穿插热点话题和实际案例，增强可读性。必须使用最新热点案例（优先使用最近1-2个月内的热门AI工具或事件），避免使用过时案例。**科技类文章尤其要注意时效性，如使用Clawdbot、Grok-3、DeepSeek等最新AI工具作为案例，而非ChatGPT等早期工具。案例需包含具体时间点、数据支撑和具体场景，如"上周发布的Clawdbot V3版本"或"某机构测试显示转化率提升40%"。
   - 贴近生活，像拉家常
   - 设计结尾：要么恍然大悟，要么意味深长，不说透
   - **典故与引用规范（重要）**：
     - 不使用古典文言典故（庄子、论语、诗经、唐诗宋词等）
     - 优先使用近现代国内外经典文学作品或电影作品（如《活着》《天堂电影院》《甲方乙方》《布达佩斯大饭店》《百年孤独》《存在与虚无》等）
     - 引用时：先引原文或原场景，再解构其与当下话题的内在联系，使典故服务于文章观点，而非装饰性堆砌
     - 示例（引用《活着》）：不直接说"福贵经历了很多苦难"，而是引出福贵牵着老牛走远这个画面，解构为"一个人活到最后，身边只剩下和自己一样老的东西，这才是活着的真相"——使引用与文章核心观点形成有机勾连
     - 引用数量：每篇文章控制在1-2处，宁精勿滥

3. **优化润色**：
   - 检查段落逻辑是否连贯，是否在"绕"中越说越深
   - 确保观点清晰、论证有力但不直接说透
   - 调整语言表达，符合刘震云风格
   - 控制节奏，重复是艺术不是啰嗦

4. **生成爆款标题**（文章完成后）：
   - 基于完成的文章内容，提炼核心亮点和关键信息
   - 确定标题：**朴实有味的标题**，符合刘震云的标题风格，不花哨但有深意
   - 标题生成参考：见 [references/viral-title-best-practices.md](references/viral-title-best-practices.md)
     - **核心内容**：爆款标题特质、设计要素、不同赛道风格（科技/情感/职场/教育/理财/健康/美食/旅游/时尚）
     - **生成方法**：5步法生成流程、6种标题模板（数字型/疑问句/对比句/金句型/感叹句/痛点型）
     - **高级技巧**：多维度组合、情绪引导、时效性、社交属性
     - **实战案例**：失败案例分析和AB测试对比实例
   - 生成多个标题备选（3-5个），选择最优标题
   - 确保标题与文章内容高度匹配，避免标题党

**步骤3：事实信息筛查与验证（重要）**
文章初稿完成后，**必须**对所有事实性内容进行准确性验证，防止将错误信息写入最终文章。

1. **筛查范围**（逐一核查以下类型的事实信息）：
   - **时间信息**：所有出现的时间点（年份、月份、日期、时间段）必须准确。数据必须是当前时间（获取对话时间戳）或能获取最新数据的最近时间（精确到月份）
   - **数据信息**：所有出现的数字（统计数据、百分比、金额、排名等）必须有可靠来源支撑
   - **第三方引用**：所有引用的他人言论、著作内容、报道内容必须核实原文
   - **人物/机构信息**：提及的人物姓名、头衔、机构名称必须准确
   - **事件信息**：描述的历史事件、新闻事件的时间地点经过必须准确
   - **个人观点与经历**：作品中引用的个人经历、主观判断需标注为观点（而非事实）

2. **验证方式**：
   - 对存疑的事实信息，使用 `WebSearch` 工具进行搜索验证
   - 优先使用权威来源（政府网站、学术机构、权威媒体）进行交叉验证
   - 同一信息至少对比 2 个不同来源，取交集确认
   - 如无法验证的信息，在文章中以"据了解""有报道称"等措辞标注不确定性
   - 如发现素材中的信息与验证结果矛盾，以验证结果为准修改文章

3. **时间敏感性处理**：
   - 所有数据类信息必须以当前时间（对话时间戳）为基准
   - 优先使用最近可获取的数据（精确到月份），标注数据截止时间
   - 示例：如当前为 2026 年 5 月，文章中使用"截至 2026 年 4 月""2026 年 Q1 数据显示"等明确标注
   - 动态数据（如股价、人口、用户数等）必须标注具体日期，避免使用"目前""现在"等模糊表述

4. **筛查结果输出**：
   - 文章完成后，在 Markdown 文件末尾和 HTML 文件末尾附加**事实筛查结果**
   - 格式如下：

   ```
   ---
   ## 事实筛查结果

   | 序号 | 信息类型 | 原文内容 | 验证结果 | 来源 |
   |------|---------|---------|---------|------|
   | 1 | 时间 | "2025年3月发布" | ✅ 准确 | 来源A、来源B |
   | 2 | 数据 | "增长40%" | ⚠️ 修正为38% | 来源C |
   | 3 | 引用 | "某人曾说过..." | ✅ 准确 | 原文出处 |
   | ... | ... | ... | ... | ... |

   **筛查结论**：共核查 X 条事实信息，其中 Y 条准确，Z 条修正，W 条无法验证（已标注）。
   ```

**步骤4：自动排版优化**
智能体将为文章进行专业的公众号排版优化：

1. **结构优化**：
   - 添加吸引眼球的标题（加大加粗）
   - 设置简短摘要或引语（用引用样式）
   - 合理划分段落（每段3-5句话，避免过长）
   - 使用小标题分割内容（三级或四级标题）
   - **注意：不使用分隔线**

2. **视觉优化**：
   - 重点内容加粗（关键词、核心观点）
   - 金句和重要数据用引用样式突出
   - 关键数字使用特殊标记（如>100万）
   - 列表化要点（使用有序或无序列表）
   - 长段落拆分为短句，提升可读性

3. **互动设计**：
   - 文章结尾添加引导互动的话语
   - 设置问题引导读者留言
   - 添加"点赞、在看、转发"引导

4. **排版样式示例**：
   ```
   [大标题：加粗、24号字]

   [摘要/引语：引用样式、16号字、灰色]

   [小标题1：加粗、18号字]

   [正文段落：16号字、行间距1.75]
   **重点内容加粗**，普通内容保持原样。

   > 金句或重要观点用引用样式

   - 要点1：使用列表
   - 要点2：清晰明了
   - 要点3：易于阅读

   [小标题2：加粗、18号字]

   [正文段落：16号字、行间距1.75]

   [结尾：引导互动]
   您怎么看？欢迎在评论区留言讨论~
   点赞和"在看"，是对我最好的支持👇
   ```

**排版优化参考**：见 [references/formatting-best-practices.md](references/formatting-best-practices.md)（公众号排版的专业技巧和样式指南）

2. **样式优化**：
   - 所有样式内联，确保复制到公众号后保持效果

3. **调用方式**：

**步骤6：质量检查与最终交付**

1. **文案检查**：
   - 标题朴实有味（评分 1-10）
   - 正文流畅度和可读性
   - 情绪共鸣强度（冷幽默是否到位）
   - 行动指令清晰度
   - 检查文案是否存在常识性与合理性错误
   - **案例时效性检查**：确保使用最新热点案例（1-2个月内）
   - **案例完整性检查**：包含具体时间点、数据支撑和应用场景

2. **事实筛查验证**（重要）：
   - 确认所有时间信息准确，数据标注了截止时间
   - 确认所有数据有可靠来源，已进行多源交叉验证
   - 确认第三方引用已核实原文
   - 确认无法验证的信息已在文中标注不确定性
   - 确认文章末尾已附加事实筛查结果表格

3. **排版检查**：
   - 检查是否有重复分隔线
   - 检查段落长度是否合适
   - 检查重点内容是否突出

4. **HTML检查**：
   - 在浏览器中预览HTML，确认排版正确
   - 确认可以正常复制粘贴到公众号编辑器

5. **优化建议**：
   - 列出可改进点
   - 提供优化方案，并自行进行优化修改
   - 给出最终发布建议
