# 商机管理数据分析报告生成器

## 技能目标

基于企业快管商机管理Excel数据，自动生成专业、深入、可操作的数据分析报告，帮助业务决策。

## 触发条件

以下情况应启用本技能：
- 用户上传Excel文件，要求生成商机管理数据分析报告
- 用户提到"商机分析"、"线索分析"、"销售数据分析"等关键词
- 用户需要月度/季度商机管理报告
- 用户说"分析一下商机数据"、"生成分析报告"等

## 输入数据格式

Excel文件应包含以下字段（字段名可略有不同的表述）：
- **客户名称** - 企业名称
- **申请日期** - 商机创建时间
- **试用结束日期** - 试用期结束时间（可选）
- **客户来源** - 渠道来源（企业微信/注册/小程序/钉钉等）
- **手机号** - 联系方式（用于计算数据完整度）
- **需求** - 客户需求描述（用于计算数据完整度）
- **跟进次数** - 销售跟进次数
- **客户状态** - 当前状态（试用/线索/成交等）

## 执行流程

### 第一步：数据读取与验证

1. 使用pandas读取Excel文件
2. 自动识别字段映射（处理不同命名习惯）
3. 输出数据概况：
   - 总记录数
   - 时间范围
   - 字段完整度
   - 数据质量评分

### 第二步：核心指标计算

计算以下核心指标体系：

**获客指标**
- 总商机量、月度新增趋势
- 渠道分布及占比
- 月度环比增长率

**转化指标**
- 试用转化率（试用数/总数）
- 各渠道试用转化率
- 转化趋势（按月）

**跟进指标**
- 平均跟进次数、中位数
- 零跟进率（跟进次数=0的占比）
- 各渠道跟进率（跟进次数>0的占比）
- 深度跟进率（跟进次数≥2的占比）

**数据质量指标**
- 手机号填写率
- 需求填写率
- 各字段完整度评分

### 第三步：深度洞察分析

**渠道效能矩阵**
- 高量高质：量大且转化好（重点维护）
- 高量低质：量大但转化差（需优化）
- 低量高质：量小但转化好（需放量）
- 低量低质：量大但转化差（考虑淘汰）

**跟进质量诊断**
- 零跟进商机清单
- 各渠道跟进率对比
- 跟进次数分布（0次/1次/2次/3次+）

**趋势分析**
- 月度商机量趋势
- 月度转化率趋势
- 月度跟进率趋势

**数据质量评估**
- 各字段完整度
- 数据质量评分
- 改进建议

### 第四步：生成报告

**必须生成两个文件：**

1. **HTML可视化报告**（`商机管理_月度分析报告_YYYY-MM.html`）
   - 使用ECharts图表库
   - 包含封面、KPI卡片、多个可视化图表
   - 配色方案：深蓝（#1a237e）+ 蓝色（#1565c0）+ 橙色（#ff6f00）
   - 响应式布局

   必含图表：
   - 渠道分布饼图
   - 月度趋势柱状图
   - 渠道效能矩阵散点图
   - 跟进次数分布柱状图
   - 月度转化率趋势折线图
   - 渠道跟进率对比条形图
   - 零跟进商机漏斗图

2. **Markdown文字报告**（`商机管理_月度分析报告_YYYY-MM.md`）
   - 执行摘要（核心发现 + 优先行动）
   - 获客分析
   - 跟进质量分析
   - 转化效率分析
   - 渠道效能评估
   - 数据质量诊断
   - 改进建议

### 第五步：输出与交付

1. 将HTML和MD文件保存到 `output/` 目录
2. 使用 `preview_url` 工具预览HTML报告
3. 使用 `deliver_attachments` 工具交付两个文件
4. 在回复中总结核心发现和优先行动建议

## 分析要点

### 必须深入分析的问题

1. **为什么某些渠道转化率高但跟进率低？**
   - 企业微信：量大但跟进不足
   - 分析是否销售精力分配问题

2. **为什么某些渠道跟进率高但转化率低？**
   - 小程序/APP：跟进积极但未转化
   - 分析是否产品流程问题

3. **试用率为何下降？**
   - 按月对比试用率变化
   - 分析是否新渠道质量下降

4. **零跟进商机为何这么多？**
   - 计算零跟进占比
   - 分析是否分配机制问题

### 必须给出的 actionable 建议

每个问题必须对应：
- 具体问题描述
- 影响量化（金额/数量）
- 建议行动方案
- 预期改善效果
- 优先级（P0/P1/P2）

## 注意事项

- Excel字段名可能不标准，需要模糊匹配
- 日期字段需要统一转换为datetime格式
- 缺失值需要合理处理（不删除，标记缺失）
- 图表必须有标题、标签、数据标签
- HTML报告必须可在浏览器中直接打开
- 分析报告必须包含具体数字，不能只有"较高""较低"等模糊表述
- 建议必须可操作，不能只有"加强管理"等空话
