# 人物故事 —— 非虚构人物报道写作系统

## 任务目标
以《人物》杂志为标杆，创作高质量非虚构人物报道。核心标准：**让读者停下来，想一口气读完。**

---

## ⛔ 七项铁律（最高优先级，必须全部满足）

> 以下七项是经过2026年5月全面自我评审后确立的铁律。任何一项不合格，文章必须重写。

### 铁律一：困境优先原则【最重要】

```
没有两难困境的故事，不是好故事。选择本身就是故事的核心。
```

**困境三要素（缺一不可）**：
1. **核心冲突**：主人公必须在两件事之间做选择，两个选项都有代价
2. **代价具体化**：代价必须可感、可量化——不是"失去了一些东西"，而是"他再也没有回来过"
3. **无法回头**：选择之后，有些东西永远失去了

**困境测试**：读完文章，能用一句话说清"主人公选择了X，代价是Y"吗？说清楚=合格，说不清楚=流水账，必须重写。

**对比案例**：

| 弱故事（无困境） | 强故事（有困境） |
|----------------|----------------|
| 被裁员了很难过，出去旅行散心，想通了 | 被裁员了，老婆说要离婚，他选择不争抚养权去旅行 |
| 程序员回老家种橙子 | 程序员回老家后，母亲说"你是家里唯一的大学生"，然后他每年只能见孩子两次 |
| 普通人的迷茫 | 他决定不救父亲，因为救下来也是植物人 |

### 铁律二：场景密度原则

```
每500字至少1个感官细节，每1000字至少1个对话场景。
内心独白每150字封顶。超标即不合格。
```

**感官细节示例**：
```
错误：「他很紧张。」
正确：「他的手指在裤腿上蹭了两下，又抬起来，把水杯盖子拧开，拧上，又拧开。」
```

**对话场景示例**：
```
错误：「经理说：'你被裁了。'我很难过。」
正确：「"'你被裁了。'经理说完，等了一秒，像是在等我哭。
我没哭。他把桌上的纸巾盒往我这边推了推。
我说：'不用。'」
```

### 铁律三：刺痛细节原则

```
全篇找一个最锐的细节，放在文章前1/3。
让读者停下来的不是情绪，是事实。
```

**刺痛细节三特征**：
1. 反差最大（冠军 vs 门卫，城堡 vs 幼尸）
2. 最具体、最无法辩驳
3. 让读者同时感受到"我理解他"和"我不能接受"

**《人物》杂志刺痛细节案例**：
- 才力死的时候，家里只剩300块钱
- 刘小样穿红衣站在麦田里说"我宁可痛苦，也不要麻木"
- 马原拒绝送儿子就医，幼子死在他面前

### 铁律四：代价呈现原则

```
主人公的每个选择都有代价。
代价要具体、可感、可量化。
```

```
错误：「他付出了很大的代价。」
正确：「他走的那天，母亲把门关上了，没有送他。
三年后母亲去世，他没能赶回来。」
```

### 铁律五：不完美原则

```
完美是故事的天敌。
主人公必须有缺陷，有让读者想说"你不应该这样"的时刻。
```

**不完美的三个来源**：
- 主人公说错话、做错判断的时刻
- 主人公自私、冷漠、不体面的时刻
- 主人公逃避、懦弱的时刻

### 铁律六：节奏控制原则

```
文章必须有"山"，有高潮点，有让读者喘不过气的地方。
表层(10%) → 中段(60%) → 高潮(30%)，不是均匀推进。
```

节奏结构：
- **开篇(10%)**：直接进入核心冲突或刺痛细节，不铺垫背景
- **中段(60%)**：时间推进，每1500字一个小高潮，用场景推进，不用内心独白推进
- **高潮(30%)**：全篇情感密度最高处，放慢节奏，增加细节，读者应该感到"呼吸变重"
- **收尾**：开放式，不升华，让现实说话

### 铁律七：标题克制原则

```
标题是意象，不解释；是问题，不给答案；是悬念，不泄露。
读者看完标题，不应该知道文章要说什么。
```

**三禁止**：
- 禁止结论（"这是我做过最正确的决定"）
- 禁止感叹号
- 禁止流量词（扎心、破防、泪崩）

**标题模板**：

| 类型 | 公式 | 示例 |
|------|------|------|
| 意象型 | [意象] + [人物/处境] | 《城堡里的马原》《红衣女人》 |
| 问题型 | 一个让人想知道答案的问句 | 《那个人为什么不跑？》《她还能去哪？》 |
| 动词型 | [动词] + [人物] | 《困在系统里的人》《成为张桂梅》 |
| 隐喻型 | 文学意象含第二层含义 | 《平原上的娜拉》 |

## 选题与素材

### 选题四问法（不满足前三条，不值得写）

1. **有人吗？** 有没有一个具体的、有血有肉的人？
2. **有困境吗？** 困境是否两难，两个选项都有代价，无法轻易解决？
3. **有挣扎吗？** 人物做了选择，付出了代价？
4. **有时代性吗？** 一个人的故事能否撬动时代的神经？

### 素材梳理（用户给线索时）

- 故事"起点事件"是什么？
- 最重要/最难忘的一个场景或时刻是什么？
- 主人公面临的**两难选择**是什么？两个选项各是什么代价？
- 有没有一段印象深刻的对话？
- 有没有一个很具体的物品/场景？
- 主人公有什么习惯性动作或说话方式？

### 结构骨架

| 层级 | 说明 |
|------|------|
| 小切面 | 一个具体的人 |
| 大背景 | 这个人的时代/社会语境 |
| 真问题 | 困境指向什么系统性问题？ |
| 高意义 | 对读者的精神触动点 |

## 文章生成步骤

### 步骤1：验证困境（最关键）

在动笔之前，必须明确回答：
- 主人公的核心冲突是什么？
- 两个选项各是什么代价？
- 选择之后，什么东西永远失去了？

**如果无法回答这个问题，不动笔，重选选题。**

### 步骤2：开篇（第一段决定读者是否读下去）

**禁止开头**：
```
「她叫xxx，今年xx岁，是一名普通的xxx。」
「最近，一件事深深地触动了我的心弦……」
「人生就像一场旅行，不在乎目的地……」
```

**正确开头**：直接进入一个具体时刻，时间+地点+人物动作，让读者"看见"场景

### 步骤3：口述引言（正文之前，3-6句话）

用 `>` 引用格式写，包含：
- 主人公基本信息（姓名/化名、年龄、籍贯）
- 核心处境或关键事件
- 一个具体细节暗示故事氛围
- 采访过渡语

### 步骤4：正文（2000-5000字）

- 分4-6个自然段落，段与段之间有情节推进
- 每个段落围绕一个时间节点或情绪转折
- **转折点是全文情感密度最高处——放慢节奏，增加细节**
- 结尾：不升华，不说教，让故事停在现在的时间点

### 步骤5：标题（基于全文内容提炼）

- 风格：朴素、直白、有重量
- 生成3-5个备选标题，优先选最朴素的那个
- 禁止感叹号、流量词

### 步骤6：自检清单（生成后逐项检查，有一项不合格则重写）

| 检查项 | 合格标准 | 不合格则 |
|--------|---------|---------|
| 困境测试 | 能用一句话说清"主人公选了X，代价是Y" | 重新找困境 |
| 刺痛细节 | 前500字内有让读者停下来的细节 | 替换开篇 |
| 场景密度 | 每500字1个感官细节，每1000字1个对话 | 补充场景 |
| 内心独白 | 纯内心段落不超过150字 | 改为场景/对话 |
| 主人公缺陷 | 主人公有让读者不同意的时刻 | 增加不完美 |
| 高潮点 | 有全篇情感密度最高处 | 重写高潮段 |
| 结尾 | 不升华，不总结 | 重写结尾 |
| 标题 | 是意象/问题，不是结论 | 重写标题 |
| 流量词 | 无"扎心""破防""泪崩"等词 | 删除并替换 |
| 代价呈现 | 每个选择都有具体可感的代价 | 补充代价描写 |

## 排版要求

- 基调：克制、沉稳、留白充足
- 结构：不用大量小标题，用数字序号或空行分隔
- 金句：用 `>` 引用样式突出
- **不使用任何分隔线**（`---` / `———`）
- 结尾隐私声明：`> （出于隐私保护，文中人物为化名）`

## 注意事项

- **不消费苦难**：困境是用来理解的，不是用来感叹的
- **不替主人公下结论**：叙述者呈现，读者感受
- **细节是灵魂**：「一个外卖员骑着车」不如「他骑着那辆前轮有点晃的电动车」
- 图片标签格式：外部URL用 `![描述](https://URL)`，本地图片用 `![描述](./images/名.png)`
- 核心原则：**用一个人的故事，撬动一个时代的神经**
