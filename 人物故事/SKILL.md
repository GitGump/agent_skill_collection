# 人物故事 —— 非虚构人物报道写作系统

## 任务目标
本 Skill 用于：以《人物》杂志为标杆，创作高质量非虚构人物报道，从选题策划、深度采访、故事创作、事实核查到排版发布的一站式服务。

能力包含：
1. 挖掘具有传播力的人物故事选题
2. 以《人物》杂志风格，创作 **2000～5000 字**的非虚构人物报道
3. **事实信息筛查**：对素材中所有事实性内容进行准确性验证，防止错误信息写入
4. 自动完成文章排版（适合公众号展示，克制、留白、沉浸）
5. 生成可直接复制到公众号的 HTML 文件
6. 提供发布操作指引

触发条件：
- 用户说「帮我写一篇人物故事」「以人物杂志风格写一篇」
- 用户提供一个真实人物线索/采访素材/生活观察
- 用户说「写一篇人物报道」「写一个特稿」「写非虚构人物」
- 用户提到社会热点人物、争议人物、命运转折者、边缘群体、城漂、乡村等主题

## 核心方法论（必读）

《人物》杂志方法论是中国非虚构人物报道的标杆，其核心可归纳为以下六步：

### 方法论一：钻石选题法
```
选题公式 = 1个具体的人 × 1个时代性问题 × 1个普遍情感
```
- **不要写"外卖行业的问题"** -> 要写"一个骑手今天迟到了5分钟被扣了钱"
- **不要写"农村女性困境"** -> 要写"刘小样穿红衣站在麦田里说'我宁可痛苦，也不要麻木'"
- **不要写"直播带货乱象"** -> 要写"辛巴直播间里一场假燕窝风波"
- **不要写"知识分子问题"** -> 要写"马原带着儿子住进城堡，拒绝现代医学"

### 方法论二：刺痛细节法
```
好细节 = 意料之外 + 情理之中 + 无法忘怀
```
操作步骤：
1. 采访中收集所有细节（不预设哪个重要）
2. 找到那个"反差最大"的细节（冠军->门卫、红衣->灰村、城堡->幼尸）
3. 把这个细节放在最关键的位置（开头或转折点）
4. **不多解释**，让细节自己说话

### 方法论三：冰山结构法
```
文章结构 = 表层故事(20%) + 暗线问题(40%) + 时代隐喻(40%)
```
- 表层故事：吸引读者走进来（一个作家隐居的故事）
- 暗线问题：让读者停下来思考（父权控制下的家庭悲剧）
- 时代隐喻：让读者走出去还在想（理想主义如何走向极权）

### 方法论四：参与式采访法
```
采访原则 = 降低存在感 + 深度嵌入 + 多元视角
```
1. **跟受访者一起生活**——吃他们吃的东西，走他们走的路
2. **采访至少5类人**——本人、家人、朋友、对手、旁观者
3. **问"那天具体发生了什么"**——而不是"你怎么看"
4. **二次回访**——《平原上的娜拉》作者花了近一年寻找失踪20年的刘小样

### 方法论五：零评判呈现法
```
呈现原则 = 事实最大化 + 评判最小化 + 留白最大化
```
- 不说"这很可悲" -> 而是写出"他死的那天，家里只有300元"
- 不说"这很不公平" -> 而是写出"系统给他派了3单，路线互相矛盾"
- 不说"她很勇敢" -> 而是写出"她穿着红衣站在村里，所有人都看着她"

### 方法论六：出圈传播公式
```
刷屏 = 共情点(个人命运) + 争议点(社会分歧) + 金句传播
```
- **共情点**：每个人都能在故事中看到自己或身边的人
- **争议点**：没有标准答案，引发正反方讨论
- **金句传播**："我宁可痛苦，也不要麻木" / "困在系统里" / "城堡"

## 操作步骤

### 步骤1：选题与采访素材梳理

基于用户提供的线索/主题：

1. **判断主题适配性**：
   - 是否属于《人物》杂志核心选题范围：争议人物、命运转折者、边缘群体、乡土中国、城市漂泊、生死、疾病、职场困境、社会热点
   - 评估故事中是否有真实的人物、具体的处境、情感的流动
   - 确认：是否触及**时代神经**——这是最核心的价值判断

2. **提炼核心叙事角度**：
   - 这个故事从谁的视角讲？（第一人称叙述者 or 第三人称）
   - 这个人面对的核心困境是什么？
   - 故事最打动人的细节/时刻在哪里？
   - 不追求猎奇，而是挖掘**普通处境中的非凡体验**

3. **建立故事结构骨架**（四层递进）：
   - **小切面**：一个具体的人
   - **大背景**：这个人所处的时代/社会语境
   - **真问题**：这个人的困境指向了什么系统性问题
   - **高意义**：故事对读者的精神触动点

4. **建立叙事结构**：
   - 开场：一个具体的场景、细节或动作，直接抛入故事中
   - 背景铺垫：主人公是谁，她/他从何处来
   - 核心事件：改变发生的那个节点
   - 情感流动：内心世界的挣扎与选择
   - 余韵结尾：不说教，不升华，留白给读者

**选题参考**：见 [references/story-topic-guide.md](references/story-topic-guide.md)

### 步骤2：事实信息筛查（⚠️ 写作前必须执行）

**在正式写作前，必须对用户提供的所有素材中的事实性内容进行准确性验证**，防止将错误信息写入文章。

#### 2.1 获取当前时间
- 在开始筛查前，**先获取对话的当前时间戳**（精确到分钟）
- 使用 Bash 命令获取：`date "+%Y年%m月%d日 %H:%M"`
- 此时间作为"数据基准时间"，文章中所有数据必须是此时间或更早的最新数据（精确到月份）

#### 2.2 事实筛查范围
对以下类型的事实性内容逐一检查：

| 类别 | 检查内容 | 核查方式 |
|------|---------|---------|
| **时间信息** | 具体日期、年份、事件发生时间 | 比对多个来源，确认无矛盾 |
| **数据信息** | 数字、比例、金额、人数、规模 | 必须是基准时间的最新数据，必要时 WebSearch 核查 |
| **第三方引用** | 他人言论、媒体报道、官方声明 | WebSearch 验证来源真实性和原文准确性 |
| **个人经历** | 主人公述说的个人经历、履历 | 与可查证资料比对，标注无法核实的部分 |
| **背景信息** | 行业数据、社会背景、政策信息 | WebSearch 确认信息时效性（需为最近时间的数据） |

#### 2.3 筛查执行规则
1. **必须核查**：文章中出现任何具体数字、日期、引用第三方内容时，必须执行 WebSearch 验证
2. **时效性要求**：数据必须是当前时间或最近月份的数据；若素材中的数据已过时，需更新为最新数据或注明"截至XXXX年XX月"
3. **来源对比**：至少查证 2 个独立来源，来源互相矛盾时，取更权威来源或注明"存在争议"
4. **个人观点标注**：若素材中含主人公个人观点/主观陈述，不验证其"对错"，但需在文章中明确以引语或说明标注为个人视角
5. **无法核实的内容**：标记为"[未经独立核实]"，在文章注释或结尾说明

#### 2.4 筛查结果报告
完成文章写作后，必须在文章正文之后附上筛查报告，格式如下：

```markdown
---

## 📋 事实筛查报告

**基准时间**：[当前时间戳，如 2026年05月27日 10:30]

| 事实项 | 原素材内容 | 核查结果 | 核查来源 | 备注 |
|--------|-----------|---------|---------|------|
| 示例 | XXX年发生了XXX | ✅ 核实 | [来源链接/说明] | — |
| 示例 | 行业规模约XXX亿 | ⚠️ 已更新 | [新来源] | 原数据为2023年，已更新为2025年数据 |
| 示例 | 主人公述说的某经历 | ❓ 无法核实 | — | 个人陈述，已在文章中以引语呈现 |

**筛查结论**：[简短总结，如"全部核实通过"/"X项数据已更新，Y项无法独立核实已注明"]
```

- 此报告单独保存为 `事实筛查报告.md`，与文章同目录存放
- 也在控制台输出供用户查阅

### 步骤3：文章创作

以《人物》杂志写作风格创作文章：

**写作风格核心特征**：
- **叙述视角**：第三人称全知为主（区别于真实故事计划的第一人称），也可第一人称
- **语言质地**：优美、克制、有深度，不刻意煽情，情绪在细节中渗透
- **叙事节奏**：慢进、不赶，细节密度高，给读者沉浸感
- **故事姿态**：不俯视，不仰视，与笔下的人同等高度站立
- **情感原则**：不做是非对错的价值判断，触及人物的内心、悲喜、情感
- **结尾处理**：开放式结尾，让现实本身说话，不强行给出答案

**代表性叙事技法**：
- 文章开头用**口述引言**（记者手记风格）交代主人公简要信息
- 正文开篇用一个极具现场感的细节/场景切入
- 用具体动作代替情绪描述
- 穿插对话和内心独白，增加人物厚度
- 避免连续议论，让故事本身承载意义
- 文章标题：朴素有力，像一句陈述句或一个问题，不用感叹号

**口述引言（开头段落）**：
- 位于标题之后、正文之前
- 以记者采访的口吻撰写，参考《人物》杂志的引言风格
- **引言内容应包含**：
  - 主人公基本信息：姓名（化名）、年龄、籍贯/家乡
  - 职业/身份：当前或核心的职业状态
  - 故事背景：主人公所处的具体处境、面临的困境或关键事件
  - 一个具体的细节或场景暗示：让读者在引言中就能感受到故事的氛围
  - 采访过渡语：用"以下是他的讲述""本文根据她的讲述整理"等方式引入正文
- **引言篇幅**：3~6句话，比纯信息交代稍丰富，但仍保持克制
- **Markdown 格式**：使用引用语法 `>` 包裹

1. **生成文章结构**：
   - 字数：**2000~5000 字**（口述引言不计入正文字数）
   - 分 4~6 个自然段落，段与段之间有情节推进
   - 每个段落围绕一个时间节点或情绪转折

2. **撰写正文内容**：
   - 口述引言：用 `>` 引用格式写一段主人公简要信息+采访过渡语
   - 开篇：抛入具体场景（地点+时间+人物动作），3~5句内建立现场感
   - 中段：故事的主要进程，穿插细节、对话、内心独白
   - 转折点：那个改变了一切的时刻，是全文情感密度最高处
   - 收尾：带着未解的余味，不强行给出答案

3. **优化润色**：
   - 检查是否有「流量写作」套路痕迹（如「扎心了」「破防了」「泪崩」）
   - 确认细节足够具体，避免泛泛而谈
   - 检查结尾是否说得太透
   - **写作时，将事实筛查中已更新/已注明的信息准确反映到文章中**

4. **生成标题与文件名**（文章完成后）：
   - 基于全文内容，提炼核心叙事线索
   - 标题风格：**朴素、直白、有重量**，像一个陈述事实的句子
   - **文件名必须带时间戳**：格式为 `{标题}-{YYYYMMDDHHmm}`
   - 生成 3~5 个备选标题，优先选最朴素的那个
   - 避免：感叹号、过度悬念词（「震惊」「绝了」「真相」）

**标题参考**：
- 《平原上的娜拉》（意象+隐喻）
- 《外卖骑手，困在系统里》（困境+意象）
- 《城堡里的马原》（地点+人物）
- 《成为张桂梅》（动词+人物）
- 《MU5735航班上的人们》（事件+群像）

**标题生成参考**：见 [references/story-topic-guide.md](references/story-topic-guide.md)

### 步骤4：排版优化

1. **整体基调**：克制、沉稳、留白充足，不用醒目配色
2. **结构设计**：不使用大量小标题，用数字序号或空行分隔
3. **金句处理**：用引用样式（`>` 块）突出
4. **不使用任何分隔线**（`---` / `———`）
5. **结尾互动**：轻柔引导，不强迫互动
6. **隐私声明**：文章末尾加 `> （出于隐私保护，文中人物为化名）`

**排版参考**：见 [references/formatting-best-practices.md](references/formatting-best-practices.md)

## 注意事项
- **不消费苦难**：故事中的困境是用来理解的，不是用来感叹或猎奇的
- **不替主人公下结论**：叙述者呈现，读者感受，不需要代替主人公总结人生
- **细节是灵魂**：「一个外卖员骑着车」不如「他骑着那辆前轮有点晃的电动车」
- 排版不使用任何分隔线
- **事实核查优先于写作**：先核查，再写作，不颠倒顺序
- **数据时效性**：所有数据必须标注时间，过时数据需更新或注明
- 核心原则：**用一个人的故事，撬动一个时代的神经**

## 使用示例

**示例1：争议人物主题**
- 用户输入：「帮我写一个关于某作家以爱为名控制家人的故事素材」
- 执行：提炼出人物故事框架，以《城堡里的马原》风格处理，不评判，只呈现
- 关键处理：找到那个刺穿人心的细节（城堡->幼尸），把判断权交给读者

**示例2：社会热点人物**
- 用户输入：「写一个被困在算法里的外卖骑手的故事」
- 执行：以第三人称叙事，开篇进入骑手送餐的具体场景，揭示系统性问题
- 关键处理：不骂平台，而是让系统逻辑和骑手困境自行碰撞

**示例3：命运转折人物**
- 用户输入：「写一个农村女性在觉醒与现实之间挣扎的故事」
- 执行：以刘小样为参照系，抓住核心意象（红衣/麦田/村庄），呈现觉醒的代价
- 关键处理：不把她处理成"励志女性"，展现复杂的内心和选择的代价
