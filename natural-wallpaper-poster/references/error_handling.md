# 错误处理与通知机制

## 错误处理策略

### 重试机制配置

```python
# 重试配置参数
RETRY_CONFIG = {
    "max_retries": 3,                    # 最大重试次数
    "retry_delay": 300,                   # 重试间隔（秒），5分钟
    "exponential_backoff": True,           # 指数退避
    "retry_on_errors": [
        "网络连接失败",
        "API超时",
        "服务暂时不可用",
        "配额限制",
    ]
}
```

### 错误分类与处理

| 错误类型 | 错误码 | 处理策略 | 恢复方式 |
|----------|--------|----------|----------|
| 网络错误 | NET_001 | 立即重试 × 3 | 等待网络恢复 |
| API超时 | API_001 | 等待 5min 重试 | 降低请求频率 |
| 认证失效 | AUTH_001 | 刷新Token | 重新登录授权 |
| 图片生成失败 | IMG_001 | 更换提示词重试 | 使用备用模板 |
| 发布失败 | POST_001 | 等待 10min 重试 | 保存草稿待手动发布 |
| 水印处理失败 | WM_001 | 跳过水印继续 | 记录日志 |
| 存储空间不足 | STO_001 | 清理旧文件 | 通知用户 |

## 日志记录规范

### 日志级别

| 级别 | 使用场景 | 示例 |
|------|----------|------|
| DEBUG | 详细调试信息 | API请求参数、响应内容 |
| INFO | 正常流程信息 | 开始生成、发布成功 |
| WARNING | 警告但可继续 | 重试、备用方案 |
| ERROR | 错误需关注 | API失败、发布失败 |
| CRITICAL | 严重错误 | 账号被封、数据丢失 |

### 日志格式

```python
LOG_FORMAT = (
    "[{timestamp}] [{level}] [{module}] {message}\n"
    "    Context: {context}\n"
    "    Trace: {trace_id}"
)

# 示例
"""
[2024-01-15 08:00:01] [INFO] [wallpaper_generator] 开始生成壁纸
    Context: {"theme": "雪峰日出", "size": "iPhone16_Pro"}
    Trace: trace_20240115_080001

[2024-01-15 08:00:45] [WARNING] [wallpaper_generator] 图片生成超时
    Context: {"retry": 1, "max_retries": 3}
    Trace: trace_20240115_080001

[2024-01-15 08:05:02] [ERROR] [xiaohongshu_poster] 发布失败
    Context: {"error_code": "POST_001", "response": "rate_limit"}
    Trace: trace_20240115_080001
"""
```

## 故障恢复流程

```
┌─────────────────────────────────────────────────────────┐
│                    故障检测与恢复流程                      │
└─────────────────────────────────────────────────────────┘

1. 异常发生
   ↓
2. 记录错误日志
   ├── 错误类型
   ├── 错误详情
   ├── 上下文信息
   └── Trace ID
   ↓
3. 错误分类
   ├── 可重试错误 → 进入重试队列
   ├── 不可重试错误 → 进入告警流程
   └── 超时错误 → 进入降级策略
   ↓
4. 执行策略
   ┌─────────────────────────────────────┐
   │ 重试策略                            │
   │ • 指数退避：30s → 60s → 120s        │
   │ • 最大重试：3次                      │
   │ • 间隔通知：每次重试发送通知          │
   └─────────────────────────────────────┘
   ↓
5. 恢复/告警
   ├── 成功 → 发送成功通知，继续发布流程
   └── 失败 → 发送告警通知，保存待处理
   ↓
6. 手动干预（如需要）
   ├── 检查账号状态
   ├── 查看错误日志
   ├── 手动执行发布
   └── 记录处理结果
```

## 监控指标

| 指标 | 目标值 | 告警阈值 |
|------|--------|----------|
| 发布成功率 | ≥ 95% | < 90% |
| 平均发布耗时 | < 5分钟 | > 10分钟 |
| 重试次数 | ≤ 1次 | > 2次 |
| API可用率 | ≥ 99% | < 98% |
| 日志完整性 | 100% | < 95% |
