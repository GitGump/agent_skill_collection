# 微信公众号文案写作助手（余华风格）

## 任务目标
- 本Skill用于：快速生成高质量的微信公众号文案，从内容创作、事实核查、自动排版到HTML生成的一站式服务
- 能力包含：
  1. 检索近期热点和热门选题
  2. 以余华的写作风格创作公众号文案（字数控制在2000~5000字）
  3. 对素材中的事实性内容进行准确性验证（时间、数据、引用等），输出筛查结果
  4. 自动完成文章排版优化（适合公众号展示）
  5. 生成可直接复制到公众号的HTML文件（保留完整排版）
  6. 提供发布到公众号的操作指引
- 触发条件：用户提供创作主题/想法，说"帮我写一篇公众号文章"或类似表达

## 前置准备
- 依赖说明：scripts脚本所需的依赖包及版本
  ```
  markdown>=3.4.0
  beautifulsoup4>=4.12.0
  ```

## 输出配置
- **生成产物保存根路径**：`E:\codefile\trae\mp_article\公众号-余华`
- 所有产物（Markdown、HTML）均保存到此路径
- 若目录不存在，智能体将自动创建

**文件夹与文件命名规则（重要）**：
- 每篇文章生成时，**首先在根路径下创建专属文件夹**，文件夹名称格式为 `文章标题-YYYYMMDDHHmm`
  - 示例：`程序员学电工-202604171154`、`机器人跑得比人快了-202604171154`
  - 时间戳取当前会话开始时的人工时间，精确到分钟
- 所有产物（Markdown文件、HTML文件）统一存放于该文件夹内
- Markdown文件命名：`文章标题.md`（文件内标题不含时间戳）
  - 示例：`程序员学电工.md`
- HTML文件命名：`文章标题.html`
  - 示例：`程序员学电工.html`
- **严禁覆盖已有文件/文件夹**：若同名文件夹已存在，必须以新时间戳创建新文件夹
- **重写不覆盖**：同一主题重新生成时，旧文件夹保留，生成新文件夹（带新时间戳）

## 操作步骤

### 标准流程

**步骤1：热点选题**
基于用户提供的主题，智能体将：

1. **检索近期热点**：
   - 搜索与主题相关的近期热点事件、热门话题
   - 筛选1-3个最具传播潜力的切入点
   - 分析热点与用户主题的关联性，确定最佳切入点

**热点选题参考**：见 [references/hot-topic-search.md](references/hot-topic-search.md)

**步骤2：文章创作**
以余华的写作风格创作文章：

**余华写作风格特点**：
- **语言风格**：极简克制、冷峻疏离、不煽情但深情；带有适度的幽默感——初读时觉得好玩，读完回味又觉得意味深远，笑意里藏着悲悯
- **表达方式**：白描手法、象征隐喻、重复与变奏；偶尔在行文合适处引用典故原文片段或金句，并进行恰当的解构与阐发，用以推进叙述或佐证观点
- **结构特点**：开篇平静如水、中段暗流涌动、结尾余韵悠长
- **句式特点**：短句与长句相结合，节奏感好，可读性强；不追求全部都是短句，长句承载逻辑，短句制造力量，长短交织形成呼吸感
- **内容特点**：关注普通人命运、苦难中见温情、荒诞中见真实

1. **生成文章结构**：
   - 拟定大纲：3-5个主要章节，每个章节围绕一个核心意象展开
   - 设计开头：从一个场景/一个人/一个物件切入，平静中暗藏张力
   - 制造节奏：用短句和留白控制叙事节奏，越往后越紧凑

2. **撰写正文内容**：
   - 字数控制在2000~5000字左右（默认目标2000字，可根据用户要求调整，最多不超过5000字）
   - 保持余华的语言风格：极简、克制、冷峻
   - 用白描、象征、重复等手法，不说透但让人懂
   - 关注普通人的命运，写生活中的苦难与温情
   - **幽默感融入**：在克制叙述中偶尔插入一句出人意料的幽默，幽默要有余华式的底色——不是段子，是苦涩里的一声轻笑；让读者初读觉得好玩，细品觉得悲凉，回味觉得深远
   - **文学/影视引用**：在正文合适处偶尔穿插近现代国内外经典文学作品或电影作品中的原文/台词片段，优先选择读者有较高认知度的作品（如《活着》《白鹿原》《平凡的世界》《肖申克的救赎》《霸王别姬》《海上钢琴师》《请回答1988》《人生》《追风筝的人》《小王子》《百年孤独》等），引用后**必须进行恰当解构与阐发**，将其与当下话题自然勾连，使引用服务于文章观点与叙事推进，而非生搬硬套；**严禁引用《史记》《庄子》《论语》等文言古籍**
   - 穿插热点话题和实际案例，增强可读性。必须使用最新热点案例（优先使用最近1-2个月内的热门AI工具或事件），避免使用过时案例。**科技类文章尤其要注意时效性，如使用Clawdbot、Grok-3、DeepSeek等最新AI工具作为案例，而非ChatGPT等早期工具。案例需包含具体时间点、数据支撑和具体场景，如"上周发布的Clawdbot V3版本"或"某机构测试显示转化率提升40%"。
   - 短句与长句相结合制造节奏感，留白让读者自己填补；长句负责展开逻辑与情境，短句负责收住力量
   - 设计结尾：余韵悠长，不说完，留给人想象

3. **优化润色**：
   - 检查段落逻辑是否连贯，是否在克制中传递情感
   - 确保观点清晰、情感真挚但不煽情
   - 调整语言表达，符合余华风格
   - 检查节奏：长短句搭配是否自然，是否有呼吸感，不追求全部短句
   - 检查幽默感：是否有1-3处恰到好处的幽默插入，幽默是余华式的——像是不经意间说出来的，读完才发现那句话藏着东西
   - 检查文学/影视引用：若有引用，来源是否为近现代国内外经典文学或电影，是否已进行充分解构、与当下话题有机融合，避免堆砌感；确认未引用文言古籍

4. **生成爆款标题**（文章完成后）：
   - 基于完成的文章内容，提炼核心亮点和关键信息
   - 确定标题：**简洁有力的标题**，符合余华的标题风格，不花哨但有力量
   - 标题生成参考：见 [references/viral-title-best-practices.md](references/viral-title-best-practices.md)
     - **核心内容**：爆款标题特质、设计要素、不同赛道风格（科技/情感/职场/教育/理财/健康/美食/旅游/时尚）
     - **生成方法**：5步法生成流程、6种标题模板（数字型/疑问句/对比句/金句型/感叹句/痛点型）
     - **高级技巧**：多维度组合、情绪引导、时效性、社交属性
     - **实战案例**：失败案例分析和AB测试对比实例
   - 生成多个标题备选（3-5个），选择最优标题
   - 确保标题与文章内容高度匹配，避免标题党

**步骤3：事实信息筛查**
在文章创作完成后、排版优化前，智能体必须对所有素材中的事实性内容进行准确性验证：

1. **筛查范围**（逐项核验）：
   - **时间数据**：所有时间节点（年份、月份、日期）必须精确无误；数据必须是当前会话时间戳对应的最新可用数据（精确到月份），如引用历史数据需标注数据时间
   - **量级数据**：所有数字（金额、人数、百分比、增长率等）必须有可靠来源支撑，不得凭空编造
   - **第三方引用**：所有引用的他人言论、报告结论、机构名称、文献内容必须可溯源，标注原始出处
   - **人物与事件**：涉及具体人物（姓名、身份、经历）和事件（地点、过程、结果）的信息必须核对准确
   - **个人观点与经历**：若文中以第一人称或化名讲述个人经历，需确认素材中是否包含足够细节支撑，避免事实与虚构混淆

2. **验证方式**：
   - **优先使用 WebSearch** 对存疑的事实数据进行交叉搜索验证
   - **多源对比**：对关键数据至少对比 2 个独立来源，确认数据一致性
   - **时效性判断**：确认所使用的数据是否为当前时间（对话时间戳）或能获取最新数据的时间节点；若只能获取到较早数据，须在文中注明数据截止时间（精确到月份）
   - **常识校验**：对明显违背常识的数据（如"某公司市值3亿却声称营收1000亿"）进行二次核查

3. **筛查输出**（在文章最终交付时附带）：
   - 以清单形式列出所有经过核查的事实项，每条包含：
     - 事实内容摘要
     - 原始来源/引用出处
     - 验证结果（✅ 通过 / ⚠️ 修正 / ⏳ 标注时间）
   - 若发现错误，在清单中注明"已修正为：[正确内容]"
   - 若数据存在时效性限制，在清单中注明数据截至时间
   - 筛查结果附加在文章 Markdown 文件末尾（以 `## 事实核查清单` 为标题），并在 HTML 文件中以折叠区块呈现

4. **不可验证内容的处理**：
   - 对于无法通过公开渠道核验的个人经历或观点，在文中以适当措辞标注（如"据当事人回忆""其自称"等），避免以客观事实口吻陈述

**步骤4：自动排版优化**
智能体将为文章进行专业的公众号排版优化：

1. **结构优化**：
   - 添加吸引眼球的标题（加大加粗）
   - 设置简短摘要或引语（用引用样式）
   - 合理划分段落（每段3-5句话，避免过长）
   - 使用小标题分割内容（三级或四级标题）
   - **注意：不使用分隔线**

2. **视觉优化**：
   - 重点内容加粗（关键词、核心观点）
   - 金句和重要数据用引用样式突出
   - 关键数字使用特殊标记（如>100万）
   - 列表化要点（使用有序或无序列表）
   - 长段落拆分为短句，提升可读性

3. **互动设计**：
   - 文章结尾添加引导互动的话语
   - 设置问题引导读者留言
   - 添加"点赞、在看、转发"引导

4. **排版样式示例**：
   ```
   [大标题：加粗、24号字]

   [摘要/引语：引用样式、16号字、灰色]

   [小标题1：加粗、18号字]

   [正文段落：16号字、行间距1.75]
   **重点内容加粗**，普通内容保持原样。

   > 金句或重要观点用引用样式

   - 要点1：使用列表
   - 要点2：清晰明了
   - 要点3：易于阅读

   [小标题2：加粗、18号字]

   [正文段落：16号字、行间距1.75]

   [结尾：引导互动]
   您怎么看？欢迎在评论区留言讨论~
   点赞和"在看"，是对我最好的支持👇
   ```

**排版优化参考**：见 [references/formatting-best-practices.md](references/formatting-best-practices.md)（公众号排版的专业技巧和样式指南）

**步骤5：生成HTML文件**
智能体将调用脚本生成可直接复制的HTML文件：

1. **脚本调用**：
   - 调用 `scripts/generate_html.py` 处理Markdown格式文章
   - 生成包含完整内联样式的HTML文件
   - HTML文件可在浏览器中直接打开

2. **样式优化**：
   - 所有样式内联，确保复制到公众号后保持效果
   - 标题、段落、引用、列表等元素都有对应样式
   - 事实核查清单以折叠区块（`<details>`）呈现

**步骤6：质量检查与最终交付**

1. **文案检查**：
   - 标题简洁有力（评分 1-10）
   - 正文流畅度和可读性
   - 情绪共鸣强度（克制中见深情）
   - 行动指令清晰度
   - 检查文案是否存在常识性与合理性错误
   - **字数达标检查**：确保全文字数在2000~5000字范围内
   - **节奏感检查**：长短句是否搭配合理，有呼吸感，可读性强
   - **幽默感检查**：文章中是否有1-3处余华式幽默，效果是否自然不生硬
   - **典故引用检查**：若有文学/影视引用，来源是否为近现代国内外经典作品，解构是否到位，是否服务于观点而非堆砌；确认未引用文言古籍
   - **案例时效性检查**：确保使用最新热点案例（1-2个月内）
   - **案例完整性检查**：包含具体时间点、数据支撑和应用场景

2. **事实核查复查**：
   - 复核步骤3中的事实核查清单是否完整
   - 确认所有修正已在正文中正确应用
   - 确认数据时效性标注（精确到月份）

3. **排版检查**：
   - 检查是否有重复分隔线
   - 检查段落长度是否合适
   - 检查重点内容是否突出

4. **HTML检查**：
   - 在浏览器中预览HTML，确认排版正确
   - 检查HTML文件大小（建议<5MB）
   - 确认可以正常复制粘贴到公众号编辑器

5. **优化建议**：
   - 列出可改进点
   - 提供优化方案，并自行进行优化修改
   - 给出最终发布建议

**质量检查清单**：
- [ ] HTML文件已正确生成
- [ ] 在浏览器中预览HTML，确认排版正确显示
- [ ] HTML文件大小合理（建议<5MB）
- [ ] 可以正常复制粘贴到公众号编辑器
- [ ] 事实核查清单已附加（Markdown末尾 + HTML折叠区块）
- [ ] 所有数据均有来源标注，时效性已标注

**最终产物清单**：
- ✅ `[文章标题-YYYYMMDDHHmm]/`（专属文件夹）
  - ✅ `[文章标题-YYYYMMDDHHmm]/文章标题.html` - 可直接复制到公众号编辑器的HTML文件
  - ✅ `[文章标题-YYYYMMDDHHmm]/文章标题.md` - Markdown源文件（含事实核查清单）

## 注意事项
- 文章创作必须严格遵循余华的写作风格
- 排版时不使用任何分隔线（包括"———"、"—"等符号）
- **事实信息筛查为强制步骤**：所有写入文章的时间、数据、引用、事件信息必须经过来源验证，未经验证的事实不得写入正文
- **数据时效性要求**：必须使用当前会话时间戳对应的最新可用数据（精确到月份），旧数据须标注截至时间
- **筛查结果交付**：文章Markdown末尾须附加 `## 事实核查清单`，HTML中以折叠区块呈现
- **产物存放规则**：每次生成均在根路径下创建专属文件夹（格式：`文章标题-YYYYMMDDHHmm`），所有文件存放于该文件夹内，严禁覆盖已有文件夹
- **文学/影视引用规则**：只引用近现代国内外经典文学或电影作品，严禁引用《史记》《庄子》《论语》等文言古籍
- 仅在需要时读取参考，保持上下文简洁
- 当操作脆弱或需强一致性时，优先调用脚本执行并进行结果校验
- 充分利用智能体的语言理解和生成能力，避免为简单任务编写脚本
